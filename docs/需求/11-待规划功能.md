## P0 — MCP 服务器

**目标**：使 Claude Desktop、Cursor、Windsurf、Zed 等 MCP 客户端可通过自然语言调用 Gate 管理代理。

**接入配置**：
```jsonc
{ "mcpServers": { "gate": { "command": "gate", "args": ["mcp"] } } }
```

**协议**：JSON-RPC 2.0 over stdio（标准 MCP 协议）

**MCP 工具列表**：

| 工具名 | 参数 | 说明 |
|--------|------|------|
| `set_proxy` | `proxy: string, tools?: string[]` | 设置全局代理，可选同时配置工具 |
| `clear_proxy` | `tools?: string[], all?: bool` | 清除代理 |
| `get_status` | 无 | 返回当前完整状态 JSON |
| `set_tool_proxy` | `toolName: string, proxy: string` | 设置单工具代理 |
| `list_tools` | `installedOnly?: bool, category?: string` | 列出支持的工具 |
| `list_presets` | 无 | 列出所有预设 |
| `save_preset` | `name: string, desc?: string` | 保存当前配置为预设 |
| `load_preset` | `name: string` | 加载预设 |
| `test_proxy` | `proxy?: string, url?: string` | 测试代理连通性 |

**实现步骤**：
1. `Gate.Core/Mcp/McpServer.cs`：实现 JSON-RPC 2.0 over stdio 循环
   - 每次从 stdin 读取一行 JSON，解析为 `{ "method", "params", "id" }` 结构
   - 分发到对应工具处理函数，结果写回 stdout
   - 未知 method → 返回 JSON-RPC 错误码 `-32601`（Method not found）
   - 参数缺失/类型错误 → 返回错误码 `-32602`（Invalid params）
   - 内部异常 → 返回错误码 `-32603`（Internal error），不崩溃退出
2. 工具列表由 `ToolRegistry` 动态驱动，不硬编码
3. `Gate.CLI/Commands/McpCommand.cs`：实现 `gate mcp` 入口
4. SSE 模式：`gate mcp --sse --port 3001`（HTTP SSE 传输，适用于不支持 stdio 的客户端）
5. `Program.cs` 注册：`root.AddCommand(McpCommand.Build())`

**MCP 模式输出约束**：`gate mcp` 运行期间，所有日志/调试信息必须写入 stderr，stdout 只输出 JSON-RPC 响应，否则破坏协议解析。

## P1 — 功能增强

### P1.1 代理有效期

```bash
gate set http://127.0.0.1:7890 --expires 8h
gate set http://127.0.0.1:7890 --expires 1d
gate set http://127.0.0.1:7890 --expires 2026-03-18T09:00:00
```

- `config.json` 中记录 `proxyExpiresAt`（ISO8601）
- `gate` 无参启动时检查过期，自动执行 `clear`
- 状态总览显示剩余时间，<1h 时橙色警告

### P1.2 PAC 文件生成与托管

```bash
gate pac generate --proxy http://127.0.0.1:7890 --output proxy.pac
gate pac serve --port 8889        # 动态生成并托管
gate pac serve --port 8889 --file proxy.pac
```

PAC 文件格式：标准 `FindProxyForURL(url, host)` 函数，NO_PROXY 规则转为 `shExpMatch` 条件。

### P1.3 社区插件在线安装

```bash
gate plugin search clash               # 搜索社区插件（模糊匹配 id/description）
gate plugin install clash              # 从社区索引下载并安装
gate plugin install clash --force      # 覆盖已安装的同名插件
gate plugin update clash               # 更新指定插件到最新版本
gate plugin update --all               # 更新所有社区安装的插件
```

**`gate plugin search` 输出格式**：
```
名称        描述                          版本    已验证
clash       Clash 代理客户端配置           1.0.0   ✓
clash-meta  Clash Meta 分支配置            1.2.0   ✓
```

**`gate plugin install <id>`（社区安装）流程**：
1. 拉取 `index.json`（缓存 1h，超时走缓存）
2. 按 `id` 查找插件条目，未找到 → 退出码 `2`
3. 下载 `downloadUrl` 的 `tool.json`
4. SHA256 校验，不匹配 → 退出码 `1`，提示「校验失败，可能被篡改」
5. 调用本地安装逻辑（同 `gate plugin install <file>`）
6. 写入 `plugin-info.json`，`installedBy: "community"`，记录 `version` 和 `sha256`

**`gate plugin update` 流程**：
1. 检查 `plugin-info.json` 中 `installedBy` 为 `"community"`（本地安装的插件不可 update）
2. 拉取最新 `index.json`，对比 `version` 字段
3. 版本相同 → 输出「已是最新版本」；版本不同 → 自动下载、校验、覆盖安装

索引格式（`index.json`）：
```json
{
  "version": 1,
  "plugins": [
    {
      "id": "clash",
      "toolName": "clash",
      "description": "Clash 代理客户端",
      "version": "1.0.0",
      "sha256": "abc123...",
      "downloadUrl": "https://raw.githubusercontent.com/gate-community/gate-tools/main/plugins/clash/tool.json",
      "verified": true
    }
  ]
}
```

### P1.4 补全脚本自动安装

```bash
gate completion install             # 自动检测 shell，写入对应路径
gate completion install --shell zsh
```

| Shell | 写入路径 |
|-------|----------|
| bash | `~/.bash_completion.d/gate` |
| zsh | `~/.zsh/completions/_gate`（或 `$fpath[1]/_gate`） |
| fish | `~/.config/fish/completions/gate.fish` |
| pwsh | `$PROFILE` 目录下 `gate.ps1` |

### P1.5 代理配置备份与回滚

```bash
gate backup                     # 备份当前所有工具配置文件原始内容
gate backup list                # 列出备份列表（带时间戳）
gate rollback                   # 回滚到最近一次备份
gate rollback --index 2         # 回滚到第 N 份备份
gate rollback --list            # 同 backup list
```

**实现要点**：
- 备份存储于 `{DataDir}/backups/{timestamp}/`，保留最近 **5** 份，超出自动删除最旧的
- 备份内容：所有已安装工具的配置文件**原始内容**（不是 Gate 的配置，而是工具本身的配置文件）
- 备份时记录 `backup-info.json`：时间戳、包含的工具列表、配置文件路径
- 回滚时将备份中的文件内容逐一写回对应路径（原子操作）
- 不备份全局环境变量（进程级，无需备份）

### P1.6 环境变量导出脚本

```bash
gate env --export-script bash   # 输出 export HTTP_PROXY=...
gate env --export-script pwsh   # 输出 $env:HTTP_PROXY=...
gate env --export-script fish   # 输出 set -x HTTP_PROXY ...
```

Shell Hook 内部调用此命令实现持久化，也可单独用于脚本集成。

### P1.7 全局配置管理命令（gate config）

```bash
gate config list                          # 列出所有配置项及当前值
gate config get defaultTestUrl            # 读取单个配置项
gate config set defaultTestUrl https://api.github.com  # 写入配置项
gate config set testTimeoutMs 5000
gate config reset                         # 恢复所有配置为默认值
```

**实现要点**：
- 读写 `{DataDir}/config.json` 中的 `GlobalConfig` 字段
- `gate config list` 输出表格：键名 | 当前值 | 默认值 | 说明
- 值类型校验：`testTimeoutMs` 必须为正整数，`autoSaveHistory` 必须为 `true`/`false`
- `gate config set` 写入后立即生效，无需重启
- 实现文件：`ConfigCommand.cs`

### P1.8 代理连通性定时监测（gate watch）

```bash
gate watch                                # 每 60s 测试当前代理，失败时报警
gate watch --interval 30                  # 自定义间隔（秒）
gate watch --url https://api.github.com   # 自定义测试目标
gate watch --notify                       # 失败时发送系统通知（OS 通知中心）
```

**实现要点**：
- 前台运行，Ctrl+C 停止
- 每次测试输出一行：时间戳 | 状态（✓/✗）| 响应时间
- `--notify` 在 Windows 使用 toast 通知，macOS 使用 `osascript`，Linux 使用 `notify-send`
- 连续失败 3 次时输出醒目警告行

### P1.9 多代理负载均衡选择（gate pick）

```bash
gate pick http://p1,http://p2,http://p3   # 并发测试，自动选择延迟最低的代理并设置
gate pick --from-history                  # 从历史记录中选最快的
gate pick --apply                         # 选择后自动调用 gate set 设置（默认仅输出推荐）
```

**实现要点**：
- 并发测试所有候选代理（同 `gate test --compare` 逻辑复用）
- 输出排行榜后询问是否应用延迟最低的代理
- `--apply` 跳过确认直接设置

### P3.2 CI/CD 工作流

| 工作流 | 触发条件 | 产物 |
|--------|----------|------|
| `build.yml` | PR 和 push to main | 三平台（win-x64/linux-x64/osx-x64）自包含二进制 |
