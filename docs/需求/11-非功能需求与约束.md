> 所属项目：Gate · 模块化文档 v2.0

## 1. 跨平台兼容性

| 平台 | 最低版本 | 特殊说明 |
|------|----------|----------|
| Windows | 10 1903+ | `%LOCALAPPDATA%`；`--write-registry` 写 `HKCU\Internet Settings` |
| Linux | Ubuntu 20.04+ | 遵循 XDG（`$XDG_DATA_HOME/gate`） |
| macOS | 12 Monterey+ | `~/Library/Application Support/gate` |

**三平台行为一致要求**：相同命令产生相同逻辑结果，仅文件路径和系统 API 可有差异。

## 2. 性能指标

| 操作 | 目标值 |
|------|--------|
| CLI 冷启动到第一次输出 | < 500ms |
| `gate apps`（214 工具加载） | < 200ms |
| `gate preset load` | < 100ms（不含网络 I/O） |
| 代理连通性测试默认超时 | 10s（可通过 `--timeout` 配置） |
| Unity GUI 面板切换响应 | < 100ms |

---

## 3. 可靠性

- 所有文件 I/O 有 `try/catch`，失败输出明确错误，不崩溃
- 写入使用原子操作：先写 `.tmp`，成功后 `File.Replace`
- 单个工具写入失败不影响其他工具，继续处理后续工具
- `preset load` 工具写入失败跳过并在摘要报告，不回滚已成功工具
- 配置文件损坏（非法 JSON）时使用默认值，输出 WARN，不崩溃

| 类别 | 状态 | P1 计划 |

| 代理认证密码 | 明文嵌入 URL（`http://user:pass@host`） | 密钥链集成 |

## 6. 可测试性

| 测试类 | 覆盖内容 |
|--------|----------|
| `ConfigValidatorTests` | 代理格式校验（合法/非法各 10+ 用例） |
| `ProfileManagerTests` | 保存/加载/删除/重命名/设默认 CRUD |
| `ProxyHistoryTests` | 追加/去重/上限/清除 |
| `PluginManagerTests` | 安装/校验失败/卸载/覆盖规则 |
| `DeclarativeConfiguratorTests` | keyvalue/ini/json/toml 格式读写 |
| `EnvVarManagerTests` | Process 级设置和清除 |
| `ConfigMigrationTests` | export-all、import-all 跳过/覆盖 |
| `GlobalConfigManagerTests`（P1） | 默认值回退、类型校验、Reset |
| `BackupManagerTests`（P1） | 备份创建、列表、回滚、超限删旧 |

**原则**：Manager 类接受路径依赖注入，测试用临时目录；`ProxyTester` 用 mock HTTP client。

## 7. 已知约束

### 7.1 代理协议
- 工具配置文件代理仅支持 HTTP/HTTPS，不支持 socks4/socks5
- SOCKS 代理可写入环境变量，工具是否识别取决于工具本身

### 7.2 持久化
- `gate set` 仅写入进程级环境变量，新开终端不生效 → 用 `gate install-shell-hook`
- Shell Hook 修改 profile 后需 `source` 或新开终端才生效

### 7.3 平台
- `--write-registry` 仅 Windows，其他平台退出码 1
- WSL 中 Gate 以 Linux 版本运行，不自动同步 Windows 宿主机代理
- macOS SIP：Gate 不修改 `/etc/` 下系统配置

### 7.4 插件
- 当前仅支持本地文件安装（在线安装 P1 实现）
- 仅支持声明式 JSON 格式，不支持 DLL/脚本插件

### 7.5 技术
- 无统一日志框架（P2 引入 `ILogger`）
- 无配置文件 schema 版本字段（P2 添加 `schemaVersion`）
- 无跨进程锁，多终端同时写同一配置文件可能竞争（OS 行为，文档说明）

### 7.6 设计范围外（不支持）
- 系统级防火墙规则管理
- 透明代理（TProxy）
- 代理协议转换（SOCKS5 转 HTTP）
- 工具的 SSL 证书/DNS 配置
- 多用户/多租户代理隔离

## 8. P1 命令边界说明

| 命令 | 边界 |
|------|------|
| `gate watch` | 前台阻塞运行，不支持后台守护；不支持 `--json` |
| `gate watch --notify` | Linux 依赖 `notify-send`，未安装时降级为终端输出 |
| `gate pick` | 仅支持 HTTP/HTTPS 代理测试，不支持 SOCKS |
| `gate pac serve` | 仅 `127.0.0.1` 监听，不对外暴露；前台阻塞，不支持后台 |
| `gate backup` | 仅备份工具配置文件，不备份 Gate 自身数据（用 `export-all`） |
| `gate rollback` | 不影响正在运行的进程；支持 `--dry-run` 只输出文件列表 |
| `gate config set` | 仅支持 `GlobalConfig` 中已定义的键名 |
| `gate plugin search` | 需要网络；离线时退出码 1 |
| `gate plugin update` | 仅更新 `installedBy: "community"` 的插件 |
