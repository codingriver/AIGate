## 1.1 产品定位

Gate 是一款**跨平台开发代理配置管理工具**，解决开发者在复杂网络环境下（企业内网、VPN、境外代理）需要为数十种开发工具逐一配置代理的痛点。

**核心价值主张**：
- 一处配置，多工具生效（~214 个工具支持）
- 预设一键切换网络场景（公司/家庭/热点/AI开发）
- 跨平台一致体验（Windows / Linux / macOS）
- 声明式工具扩展，社区可零代码贡献
- MCP 服务器接入，AI 助手直接管理代理（P0 规划）

## 1.3 命名规范（术语表）

| 术语 | 含义 | 示例 |
|------|------|------|
| **全局代理** | 环境变量层代理（`HTTP_PROXY` / `HTTPS_PROXY` / `NO_PROXY`），作用于当前进程及子进程 | `HTTP_PROXY=http://127.0.0.1:7890` |
| **工具代理** | 写入工具自身配置文件的代理，始终生效不依赖终端会话 | git 的 `~/.gitconfig [http] proxy=...` |
| **预设（preset）** | 一组已命名的代理配置快照：全局代理值 + 各工具代理值，可保存和加载切换 | `office`、`home`、`ai-dev` |
| **声明式工具** | 通过 JSON 文件描述代理配置方式的工具，无需 C# 代码 | `tools/package-managers/npm.json` |
| **插件** | 用户/社区通过 `tool.json` 文件添加的自定义工具支持，安装到用户数据目录 | `{DataDir}/plugins/clash/tool.json` |
| **Shell Hook** | 写入 shell profile 文件的脚本，使代理设置在每次开终端时自动生效 | `eval "$(gate env --export-script bash)"` |
| **ToolDescriptor** | 描述工具代理配置方式的数据结构（JSON 或 C# 类） | `{ toolName, config, envProxy }` |
| **PlatformString** | 可以统一写法或差异化写法的跨平台路径字符串 | `"~/.npmrc"` 或 `{windows: "...", linux: "..."}` |

## 1.5 与同类工具对比

| 工具 | 方式 | Gate 的差异 |
|------|------|-------------|
| 手动配置 | 逐一修改每个工具配置文件 | Gate 批量操作，一次覆盖 214+ 工具 |
| 系统代理（IE 代理/macOS 网络偏好设置） | 只影响支持系统代理的软件 | Gate 直接写入工具配置文件，覆盖不支持系统代理的工具 |
| proxychains | 劫持进程的系统调用 | Gate 配置工具自身，更干净，无兼容性问题 |
| .env 文件 | 项目级环境变量 | Gate 是用户级工具，跨项目生效 |
