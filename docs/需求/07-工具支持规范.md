## 7.2 何时用 JSON 声明 vs C# Configurator

| 场景 | 推荐方式 | 原因 |
|------|----------|------|
| 工具配置文件为 keyvalue/ini/toml/json 格式，代理通过固定键写入 | JSON 声明 | 零代码，社区可直接贡献 |
| 工具通过环境变量读取代理 | JSON 声明（envProxy 型） | 零代码 |
| 工具配置文件为 XML（如 Maven settings.xml） | C# Configurator | XML 操作需要代码 |
| 工具有多个配置文件版本，需要条件判断 | C# Configurator | 逻辑分支无法用 JSON 表达 |
| 工具使用私有二进制配置格式 | C# Configurator | JSON 无法描述 |
| 工具代理需要编码转换（如 Base64 认证头） | C# Configurator | 值转换无法用 JSON 模板表达 |

**原则**：新工具优先尝试 JSON 声明；若无法满足需求再实现 C# Configurator。

## 7.3 ToolDescriptor JSON 完整字段

```jsonc
{
  // 必填
  "toolName": "mytool",          // 唯一标识，全小写+连字符
  
  // 可选
  "displayName": "My Tool",      // 展示名，默认同 toolName
  "category": "其他工具",          // 分类，用于 gate apps 分组

  // 配置文件型（与 envProxy 二选一，或同时支持两种）
  "config": {
    "probePaths": [
      // PlatformString：统一写法
      "~/.mytoolrc",
      // PlatformString：差异化写法
      { "windows": "%APPDATA%\\MyTool\\config", "linux": "~/.config/mytool/config", "macos": "~/.config/mytool/config" }
    ],
    "format": "keyvalue",         // keyvalue | ini | json | toml（xml 不支持，需 C#）
    "section": "http",            // ini/toml 的 section，如 [http]，可选
    "proxyLines": [
      {
        "key": "proxy",
        "value": "{proxy}",        // {proxy} 替换为实际代理地址
        "removeOnClear": true      // clear 时删除此行，默认 true
      }
    ]
  },

  // 环境变量型（可与 config 同时存在，表示工具同时支持两种方式）
  "envProxy": {
    "envVars": ["MYTOOL_PROXY", "HTTP_PROXY"]  // 按顺序设置这些环境变量
  }
}
```

## 7.4 工具命名规范

| 规则 | 说明 | 正确示例 | 错误示例 |
|------|------|----------|----------|
| 全小写 | 不含大写字母 | `mytool` | `MyTool` |
| 只用连字符 | 不用下划线或空格 | `my-tool` | `my_tool` |
| 与 CLI 命令一致 | 和用户在终端输入的命令一致 | `gh`、`kubectl`、`pip3` | `github-cli` |
| 不含版本号 | 版本无关性 | `pip` | `pip3.11` |
| 最大 32 字符 | 避免过长 | — | — |

## 7.5 工具安装检测逻辑（详细）

```csharp
bool IsInstalled(ToolDescriptor tool) {
    // 1. 检查自定义可执行路径
    var customExec = toolPaths.GetExecPath(tool.ToolName);
    if (customExec != null) return File.Exists(customExec);

    // 2. 检查自定义配置文件路径
    var customConfig = toolPaths.GetConfigPath(tool.ToolName);
    if (customConfig != null) return File.Exists(customConfig);

    // 3. 检查声明的 probePaths
    foreach (var path in tool.Config?.ProbePaths ?? []) {
        if (File.Exists(path.Resolve())) return true;
    }

    // 4. 检查 PATH 中的可执行文件
    return IsInPath(tool.ToolName);
}
```

## 7.6 配置文件写入规范

### keyvalue 格式

```ini
# 原文件
registry=https://registry.npmjs.org
proxy=http://old-proxy:8080

# gate app npm http://127.0.0.1:7890 后
registry=https://registry.npmjs.org
proxy=http://127.0.0.1:7890        # 原地更新，不新增重复行
https-proxy=http://127.0.0.1:7890  # 原来没有的键，追加到文件末尾
```

### json 格式

使用 JSON merge patch（RFC 7396）：只更新代理相关字段，不触碰其他字段。

```json
// 原文件（docker config.json）
{ "auths": { "registry.example.com": {} }, "proxies": {} }

// gate app docker http://127.0.0.1:7890 后
{
  "auths": { "registry.example.com": {} },
  "proxies": {
    "default": {
      "httpProxy": "http://127.0.0.1:7890",
      "httpsProxy": "http://127.0.0.1:7890"
    }
  }
}
```

### ini 格式

```ini
# 原文件（含 section）
[core]
repositoryformatversion = 0

[http]

# gate app git http://127.0.0.1:7890 后
[core]
repositoryformatversion = 0

[http]
proxy = http://127.0.0.1:7890   # 追加到对应 section 末尾
                                 # section 不存在时自动创建
```

**section 存在判断**：按行扫描，找到 `[{section}]` 行后插入代理行；若全文找不到该 section，追加 `\n[{section}]\n{key} = {value}` 到文件末尾。

### toml 格式

```toml
# 原文件（cargo config.toml）
[build]
target = "x86_64-unknown-linux-gnu"

# gate app cargo http://127.0.0.1:7890 后
[build]
target = "x86_64-unknown-linux-gnu"

[http]
proxy = "http://127.0.0.1:7890"  # 追加新 section（若不存在）
```

toml 格式写入使用与 ini 相同的行扫描逻辑，值需加双引号（`"{proxy}"` 模板）。

### 写入安全要求

1. 写入前检查文件可写性，不可写时返回错误，不抛异常
2. 写入使用原子操作：先写临时文件，再 rename 替换
3. 原文件不存在时，自动创建父目录（`Directory.CreateDirectory`）
4. 不修改文件的换行符风格（保留 CRLF/LF）
5. 不修改文件的编码（保留 UTF-8/GBK 等）

## 7.7 新增工具贡献流程

```
1. 确认工具代理配置方式
   └─ 查看工具官方文档或 man page

2. 创建 JSON 文件
   └─ 位置：Gate.Core/tools/{category}/{toolname}.json
   └─ 参照 7.3 节字段说明填写

3. 校验 JSON
   └─ gate plugin validate path/to/tool.json

4. 手动测试
   └─ gate app {toolname} http://127.0.0.1:7890
   └─ 检查配置文件内容是否正确
   └─ gate app {toolname} --clear
   └─ 检查配置文件代理行是否已移除

5. 提交 PR
   └─ 标题格式：feat(tools): add {toolname} support
   └─ 附带：工具官网链接 + 配置文件路径说明
```
