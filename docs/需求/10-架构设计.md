## 2. 工具加载优先级

```
低优先级 ────────────────────────────── 高优先级

① EmbeddedResource (Gate.Core/tools/**/*.json)
   随 DLL 打包，无外部文件依赖

② 特殊 C# Configurator
   同名工具时覆盖 ①（如 GitConfigurator 覆盖 git.json）

③ 用户插件 ({DataDir}/plugins/{name}/tool.json)
   同名时覆盖 ① 和 ②
   gate plugin install 安装后热注册，无需重启
```

**同名规则**：后加载覆盖先加载；覆盖时输出 DEBUG 级日志。

## 4. 代理层级与读取顺序

```
优先级（高→低）：Process > User > Machine

读取：优先取 Process，空则 User，再空则 Machine
写入：gate set       → Process 级（当前终端有效）
      gate env --write-registry → User 级（Windows 注册表）
```

---

## 5. EnvVarManager 接口规范

```csharp
public class EnvVarManager {
    // 读取生效值（Process > User > Machine）
    string? GetEffectiveProxy(string key);

    // 写入进程级（立即生效，仅当前进程及子进程）
    void SetProcess(string key, string value);
    void ClearProcess(string key);  // value=null

    // 写入用户级
    void SetUser(string key, string value);

    // 三层详情（供 gate env 展示）
    EnvLayerInfo GetAllLayers(string key);
    // EnvLayerInfo { Machine, User, Process, Effective }

    // Windows 注册表（非 Windows 抛 PlatformNotSupportedException）
    bool WriteWindowsRegistry(string http, string https, string noProxy);
}
```

## 6. ToolConfiguratorBase 接口规范

```csharp
public abstract class ToolConfiguratorBase {
    public abstract string ToolName { get; }

    // 写入代理；不可写时返回 false（不抛异常）
    public abstract bool SetProxy(string proxy);

    // 移除代理相关行；失败返回 false
    public abstract bool ClearProxy();

    // 读取当前代理值；未配置返回 null
    public abstract string? ReadProxy();

    // 工具是否已安装（三级优先级检测）
    public virtual bool IsInstalled() { ... }

    // 配置文件路径（供展示用）
    public virtual string? GetConfigFilePath() { ... }
}
```

**DeclarativeToolConfigurator**：继承基类，由 `ToolDescriptor` JSON 驱动，无需子类代码。  
**特殊 C# Configurator**（Git/Docker/Gradle/Maven）：继承基类并重写三个核心方法。

## 7. 关键设计决策

| 决策 | 原因 | 放弃的替代方案 |
|------|------|----------------|
| Gate.Core 用 netstandard2.0 | 兼容 Unity 2021+/Mono | net8.0：Unity 不兼容 |
| 工具优先 JSON 声明 | 零代码贡献，社区门槛低 | 纯 C#：每工具需 PR+编译 |
| EmbeddedResource 嵌入 JSON | 单文件发布，无外部依赖 | 外部文件：需保持目录结构 |
| GatePaths 统一管理存储路径 | 修复 Linux XDG 问题，可测试 | 路径散落：难跨平台 |
| OutputSettings 全局单例 | 一处控制所有输出模式 | 每命令各自判断：代码重复 |
| Process 级环境变量 | 不污染系统，安全可控 | User 级：影响所有新终端 |
| `System.Text.Json` Source Generator | NativeAOT 兼容（无反射） | Newtonsoft.Json：AOT 不兼容 |

---

## 8. AOT 兼容要求

`Gate.CLI` 支持 NativeAOT 单文件发布：
- `System.Text.Json` 所有序列化类型必须注册 `[JsonSerializable]`
- 禁止在发布路径使用反射（`Activator.CreateInstance` 等）
- `ToolRegistry` 的声明式工具通过 EmbeddedResource + `JsonSerializer.Deserialize<ToolDescriptor>()` 加载，已兼容 AOT
