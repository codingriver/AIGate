> 所属文档：Gate 需求文档 · [返回索引](./README.md)

## 8.1 插件存储结构

```
{DataDir}/plugins/
└── {toolName}/
    ├── tool.json         # 工具描述（格式同内置 ToolDescriptor JSON）
    └── plugin-info.json  # 插件元数据（安装时间、来源等）
```

---

## 8.2 plugin-info.json 格式

```json
{
  "toolName": "clash",
  "sourceFile": "/home/user/Downloads/clash-tool.json",
  "installedAt": "2026-03-17T22:00:00Z",
  "installedBy": "local",
  "version": "1.0.0"
}
```

`installedBy`：`"local"`（本地文件安装）或 `"community"`（社区索引安装，P1）。

> **注意**：`version` 字段在本地安装时取 `tool.json` 中的 `version` 字段（若有）；未声明时存 `""` 空字符串。`PluginInfo` C# 模型包含此字段（见 [04 §4.6](./04-数据模型.md)）。

## 8.3 插件生命周期

### 安装流程

```
gate plugin install path/to/tool.json [--overwrite]
│
├─ 1. 格式校验（见 8.4）
│      失败 → 输出所有错误，退出码 1
├─ 2. 检查目标目录是否已存在
│      存在 + 未加 --overwrite → 退出码 2，提示「已存在，加 --overwrite 覆盖」
├─ 3. 创建 {DataDir}/plugins/{toolName}/
├─ 4. 复制 tool.json
├─ 5. 写入 plugin-info.json
└─ 6. ToolRegistry.RegisterPlugin() 热注册（无需重启）
```

### 卸载流程

```
gate plugin remove {toolName} [--force]
│
├─ 1. 检查插件是否存在，不存在 → 退出码 2
├─ 2. 交互确认（--force 跳过）
├─ 3. 删除 {DataDir}/plugins/{toolName}/ 整个目录
└─ 4. ToolRegistry.UnregisterPlugin() 注销
```

---

## 8.4 校验规则

| 规则 | 错误信息示例 |
|------|-------------|
| `toolName` 非空 | `"toolName" is required` |
| `toolName` 全小写+连字符，不含空格/大写/下划线 | `"toolName" must be lowercase with hyphens only` |
| `toolName` 长度 ≤ 32 | `"toolName" exceeds 32 characters` |
| 至少含 `config` 或 `envProxy` 之一 | `Must have "config" or "envProxy"` |
| `config.format` 为合法值 | `"format" must be one of: keyvalue, ini, json, toml` |
| `config.probePaths` 非空 | `"probePaths" must not be empty` |
| `config.proxyLines` 非空 | `"proxyLines" must not be empty` |
| 每个 `proxyLine.key` 非空 | `proxyLine[0].key must not be empty` |
| `envProxy.envVars` 非空 | `"envVars" must not be empty` |

校验失败时列出**所有**错误（不在第一个错误处中止），便于一次性修复。

## 8.5 CLI 命令详细行为

```bash
# 列出已安装插件（表格）
gate plugin list
# 输出列：插件名 | 分类 | 安装时间 | 来源

# 安装
gate plugin install path/to/tool.json
gate plugin install path/to/tool.json --overwrite  # 覆盖已存在的同名插件

# 卸载
gate plugin remove clash
gate plugin remove clash --force   # 跳过确认

# 只校验不安装
gate plugin validate path/to/tool.json
# 成功：输出 "✓ tool.json is valid"，退出码 0
# 失败：输出所有错误，退出码 1

# 查看插件详情
gate plugin show clash
# 输出：tool.json 内容 + plugin-info.json 内容
```

## 8.6 ToolRegistry 插件相关接口

```csharp
// 安装时调用，立即生效
void RegisterPlugin(ToolDescriptor descriptor, PluginInfo info);

// 卸载时调用
void UnregisterPlugin(string toolName);

// 加载时（Gate 启动）扫描插件目录
void LoadPlugins(string pluginsDir);

// 查找（插件覆盖内置工具时返回插件版本）
IToolConfigurator? Get(string toolName);
```

**覆盖规则**：插件 `toolName` 与内置工具同名时，插件版本优先（高优先级）。覆盖时控制台输出 `[INFO] Plugin 'clash' overrides built-in tool`。

## 8.7 社区插件仓库规范（P1）

**仓库**：`github.com/gate-community/gate-tools`

**目录结构**：
```
gate-tools/
├── index.json           # 自动生成，CI 维护
├── plugins/
│   ├── clash/
│   │   ├── tool.json
│   │   └── README.md    # 工具说明、代理键名来源说明
│   └── v2ray/
│       ├── tool.json
│       └── README.md
└── .github/workflows/
    └── validate.yml     # PR 时自动校验所有 tool.json
```

**index.json 格式**：
```json
{
  "version": 1,
  "updatedAt": "2026-03-17T00:00:00Z",
  "plugins": [
    {
      "id": "clash",
      "toolName": "clash",
      "description": "Clash 代理客户端配置",
      "version": "1.0.0",
      "sha256": "abc123...",
      "downloadUrl": "https://raw.githubusercontent.com/.../clash/tool.json",
      "verified": true,
      "author": "community"
    }
  ]
}
```

**安全机制**：
- 每个插件有 SHA256 校验值，下载后验证
- `verified: true` 表示已由维护者审核
- 仅允许声明式 JSON，不允许可执行代码
