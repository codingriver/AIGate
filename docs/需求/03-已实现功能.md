## 3.1 全局代理管理

| 功能 | 命令 | 行为说明 |
|------|------|----------|
| 设置全局代理 | `gate set http://127.0.0.1:7890` | 将 `HTTP_PROXY` 和 `HTTPS_PROXY` 写入**当前进程**环境变量（Process 级）。新开终端不自动生效，需 `install-shell-hook` 持久化 |
| 同时配置工具 | `gate set http://... git,npm` | 先设置全局代理，再逐一写入指定工具配置文件；工具名逗号分隔，不含空格 |
| 设置前验证 | `gate set http://... --verify` | 先执行连通性测试（超时 10s），通过后再写入；失败则中止，退出码 `3` |
| 设置 NO_PROXY | `gate set http://... --no-proxy "localhost,127.0.0.1,*.local"` | 同时将 `NO_PROXY` 写入进程环境变量；值为逗号分隔主机名/IP/通配符列表 |
| 从历史选择 | `gate set --history-index 2` | 从 `history.json` 取第 N 条（1-based）地址设置；超出范围时输出错误 |
| 设置有效期 | `gate set http://... --expires 8h` | 设置代理过期时间；支持 `Nh`（小时）、`Nd`（天）、ISO8601 绝对时间；过期后 `gate` 无参启动时自动 `clear`（P1 功能，命令参数已规划） |
| 清除全局代理 | `gate clear` | 清除进程级 `HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY` |
| 清除全部代理 | `gate clear --all` | 清除全局环境变量代理，并对所有已安装且已配置代理的工具逐一清除 |
| 查看代理层级 | `gate env` | 表格展示 Machine/User/Process 三层各环境变量值，最后一行显示最终生效值 |
| 写入 Windows 注册表 | `gate env --write-registry` | 将当前代理写入 `HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings`；仅 Windows 可用 |

## 3.2 工具代理管理

**工具安装检测逻辑**（按优先级）：
1. `gate path -n <tool>` 中有自定义路径 → 使用自定义路径
2. `probePaths` 中任一配置文件存在 → 认为已安装
3. PATH 中能找到工具可执行文件 → 认为已安装

| 功能 | 命令 | 行为说明 |
|------|------|----------|
| 查看工具代理 | `gate app git` | 显示配置文件路径、代理键值、文件是否存在/可读 |
| 设置工具代理 | `gate app git http://...` | 写入配置文件代理行；文件不存在时**自动创建**；已有代理行时**原地更新**，不影响其他配置 |
| 批量设置 | `gate app git,npm,pip http://...` | 逐一处理；单个失败不中止其余；最终汇总成功/失败数 |
| 清除工具代理 | `gate app git --clear` | 从配置文件移除代理相关行；其他配置行保持不变 |
| 全部工具操作 | `gate app --all http://...` | 对所有已安装工具执行设置 |
| 排除指定工具 | `gate app --all --except git,docker http://...` | `--except` 支持逗号分隔多个工具名 |
| 列出所有工具 | `gate apps` | 按分类分组；每行：工具名、分类、已安装（✓/✗）、已配置代理（✓/✗） |
| 仅已安装工具 | `gate apps --installed` | 只显示本机检测到的工具 |
| 按分类过滤 | `gate apps --category "AI 编程助手"` | 只显示指定分类的工具，分类名与 `gate apps` 中显示的分类名一致 |

## 3.3 预设管理

| 功能 | 命令 | 行为说明 |
|------|------|----------|
| 列出预设 | `gate preset` | 表格显示：名称、描述、创建时间、是否默认、工具数 |
| 保存预设 | `gate preset save <name>` | 快照当前 Process 级 HTTP/HTTPS/NO_PROXY + 所有已配置工具的代理值 |
| 保存带描述 | `gate preset save <name> --desc "公司内网"` | 附加描述文字 |
| 加载预设 | `gate preset load <name>` | ① 设置进程级环境变量 ② 遍历 ToolConfigs 写回工具配置文件 ③ 输出摘要（成功N/跳过M） |
| 删除预设 | `gate preset del <name>` | 删除 JSON 文件；若为默认预设则同时清除默认标记 |
| 重命名预设 | `gate preset rename <old> <new>` | 重命名文件并更新 Name 字段；目标名已存在时报错 |
| 设为默认 | `gate preset set-default <name>` | 在 `config.json` 写入 `defaultPreset` 字段 |
| 查看详情 | `gate preset show <name>` | 显示预设完整内容：全局代理 + 所有工具代理键值 |
| 导出预设 | `gate preset export <name> [file]` | 输出到文件；不指定 file 时输出到 stdout |
| 导入预设 | `gate preset import <file> [--as <name>] [--overwrite]` | 读取 JSON 保存为预设；同名时默认报错，`--overwrite` 覆盖 |
| 查看预设详情 | `gate preset show <name>` | 显示预设完整内容：全局代理值 + 所有工具代理键值对 |

> **preset load 工具写入策略**：工具未安装或配置文件不可写时**跳过（不报错）**，在摘要中列出跳过的工具名。

## 3.4 连通性测试

| 功能 | 命令 | 行为说明 |
|------|------|----------|
| 测试当前代理 | `gate test` | 读取当前进程 HTTP_PROXY，发起 HTTP HEAD 请求到 `https://www.google.com`；输出：成功/失败、响应时间、HTTP 状态码 |
| 测试指定代理 | `gate test http://127.0.0.1:7890` | 使用指定代理，不受当前环境变量影响 |
| 自定义目标 | `gate test http://... --url https://api.github.com` | 向自定义 URL 测试 |
| 多代理对比 | `gate test --compare http://p1,http://p2` | 并发测试所有代理，按延迟升序排列；超时排末尾 |
| 超时配置 | `gate test --timeout 5000` | 单位毫秒，默认 10000 |

**--json 输出格式**：
```json
{
  "success": true,
  "responseTimeMs": 342,
  "httpStatusCode": 200,
  "testUrl": "https://www.google.com",
  "proxy": "http://127.0.0.1:7890",
  "errorMessage": null
}
```

## 3.5 诊断与运维

### gate doctor 检查项

| 检查项 | 通过条件 | 失败建议 |
|--------|----------|----------|
| 数据目录可写 | `{DataDir}` 存在且当前用户有读写权限 | 手动创建或修复权限 |
| 预设目录可写 | `{DataDir}/profiles/` 可写 | 同上 |
| 插件目录可写 | `{DataDir}/plugins/` 可写 | 同上 |
| 历史文件可读写 | `history.json` 可读写（不存在视为通过） | 删除损坏文件 |
| 进程级代理已设置 | `HTTP_PROXY` 或 `HTTPS_PROXY` 非空 | 运行 `gate set` |
| Shell Hook 已安装 | profile 文件中包含 gate 相关脚本 | 运行 `gate install-shell-hook` |
| 默认预设存在 | `config.json` 中 `defaultPreset` 对应文件存在 | 运行 `gate preset set-default` |
| 已安装工具配置可写 | 已安装工具的配置文件可写 | 修复文件权限 |

`--verbose`：显示全部检查项（含通过项）；普通模式只显示警告和失败项。

### gate history 行为细节

- 每次成功执行 `gate set` 时自动追加到 `history.json`
- 最多保留 **20** 条，超出时移除最旧的
- 同一地址重复使用时移到最新位置（去重）
- `gate history` 输出带编号列表，可直接用 `gate set --history-index N` 引用

### gate reset 行为

1. 交互确认（`--force` 跳过）
2. 清除当前进程级环境变量代理
3. 删除 `{DataDir}/profiles/` 全部预设文件
4. 删除 `{DataDir}/history.json`
5. 删除 `{DataDir}/tool-paths.json`
6. 删除 `{DataDir}/config.json`
7. **不删除**已安装的插件（插件需单独用 `gate plugin remove` 卸载）
8. **不修改**工具配置文件中已写入的代理（需先 `gate clear --all` 再 reset）

## 3.6 工具路径管理

**使用场景**：工具安装在非标准路径（如 NVM 管理的 Node、Homebrew 的自定义前缀、企业内部工具链）时，Gate 的自动检测无法找到。通过 `gate path` 手动指定路径。

| 功能 | 命令 | 行为说明 |
|------|------|----------|
| 查看工具路径 | `gate path -n git` | 显示：自定义可执行路径（若有）、自定义配置文件路径（若有）、自动检测结果 |
| 设置可执行路径 | `gate path -n git --exec /usr/local/bin/git` | 写入 `tool-paths.json`；此后所有涉及 git 的操作使用此路径做安装检测 |
| 设置配置文件路径 | `gate path -n git --config ~/.gitconfig-work` | 指定配置文件路径；`gate app git` 读写此文件而非自动探测的路径 |
| 清除自定义路径 | `gate path -n git --clear` | 删除 `tool-paths.json` 中对应工具的全部自定义项，恢复自动检测 |
| 列出所有自定义路径 | `gate path --list` | 表格显示所有已配置自定义路径的工具 |

**存储格式**（`tool-paths.json`）：
```json
{
  "git": {
    "execPath": "/usr/local/bin/git",
    "configPath": "~/.gitconfig-work"
  },
  "npm": {
    "configPath": "~/.npmrc-company"
  }
}
```

## 3.7 插件管理

| 功能 | 命令 | 行为说明 |
|------|------|----------|
| 列出已安装插件 | `gate plugin list` | 显示：插件名、toolName、分类、安装时间、来源文件 |
| 安装插件 | `gate plugin install path/to/tool.json` | 先校验格式 → 复制到 `{DataDir}/plugins/{toolName}/tool.json` → 热注册到 ToolRegistry |
| 卸载插件 | `gate plugin remove mytool` | 删除 `{DataDir}/plugins/mytool/` 目录 → 从 ToolRegistry 注销 |
| 校验插件 | `gate plugin validate path/to/tool.json` | 只做格式校验，不安装；输出所有错误列表（不在第一个错误处中止） |
| 查看插件详情 | `gate plugin show mytool` | 显示插件 `tool.json` + `plugin-info.json` 完整内容 |

**安装失败场景**：
- `toolName` 不符合命名规范 → 报错，提示具体不合规内容
- 缺少 `config` 和 `envProxy` → 报错
- 目标目录已存在同名插件且未指定 `--overwrite` → 报错提示已存在

## 3.8 配置迁移

**导出内容**（`gate export-all`）：
```json
{
  "version": 1,
  "exportedAt": "2026-03-17T22:00:00Z",
  "globalProxy": { "httpProxy": "http://...", "httpsProxy": "http://...", "noProxy": "localhost" },
  "globalConfig": {
    "defaultPreset": "office",
    "defaultTestUrl": "https://www.google.com",
    "testTimeoutMs": 10000,
    "autoSaveHistory": true,
    "maxHistoryCount": 20
  },
  "profiles": [ { "...Profile JSON..." } ],
  "toolPaths": { "...tool-paths.json 内容..." },
  "plugins": [
    { "toolName": "clash", "installedBy": "local", "installedAt": "2026-03-01T00:00:00Z" }
  ]
}
```

**导入行为**：
- 逐一导入 `profiles` 数组中的预设
- 同名预设：默认**跳过**（输出警告）；`--overwrite` 时覆盖
- 合并 `toolPaths`：同名工具路径：默认跳过；`--overwrite` 时覆盖
- 合并 `globalConfig`：默认跳过已有配置项；`--overwrite` 时全量覆盖
- `plugins` 字段仅记录元数据，**不自动重新安装插件**（tool.json 文件不随导出文件携带）
- 不修改当前进程级环境变量代理
- 导入完成后输出摘要：导入预设 N 个，跳过 M 个

## 3.9 Shell 集成

### install-shell-hook 写入内容

**bash/zsh**（追加到 `~/.bashrc` 或 `~/.zshrc`）：
```bash
# Gate proxy hook
if command -v gate >/dev/null 2>&1; then
  eval "$(gate env --export-script bash)"
fi
```

**fish**（写入 `~/.config/fish/conf.d/gate.fish`）：
```fish
# Gate proxy hook
if command -v gate >/dev/null
  gate env --export-script fish | source
end
```

**PowerShell**（追加到 `$PROFILE`）：
```powershell
# Gate proxy hook
if (Get-Command gate -ErrorAction SilentlyContinue) {
    gate env --export-script pwsh | Invoke-Expression
}
```

指定 `--preset <name>` 时，hook 改为执行 `gate preset load <name>` 而非导出当前代理。

`--dry-run`：只打印要写入的内容和目标文件路径，不实际修改。

### Tab 补全范围

`gate completion bash/zsh/fish/pwsh` 输出的补全脚本覆盖：
- 所有一级命令名
- 所有子命令名（如 `preset save/load/del/rename`）
- 所有 `--` 长选项名
- `gate app <toolname>` 中的工具名（动态补全）
- `gate preset load <name>` 中的预设名（动态补全）

## 3.10 全局输出模式

| 标志 | 作用范围 | 具体行为 |
|------|----------|----------|
| `--json` | 所有命令 | 所有输出为合法 JSON 对象；`gate` 状态总览输出 `{globalProxy, tools, presets}` 结构 |
| `--quiet` | 所有命令 | 抑制所有 INFO/SUCCESS 输出，只保留 ERROR；适合脚本中忽略成功消息 |
| `--no-color` | 所有命令 | 禁用 ANSI 转义码；成功/错误符号替换为 `[OK]`/`[ERR]`/`[WARN]`/`[INFO]` |
| `--plain` | 表格类命令 | 无框线无特殊符号的纯文本，列之间用制表符分隔；适合 `awk`/`cut` 处理 |

**`gate --json` 输出结构**：
```json
{
  "globalProxy": {
    "httpProxy": "http://127.0.0.1:7890",
    "httpsProxy": "http://127.0.0.1:7890",
    "noProxy": "localhost,127.0.0.1"
  },
  "configuredTools": [
    { "toolName": "git", "category": "版本控制", "proxy": "http://127.0.0.1:7890" }
  ],
  "presets": [
    { "name": "office", "isDefault": true, "toolCount": 5 }
  ],
  "activePreset": null
}
```
