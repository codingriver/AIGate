# 02 用户角色与使用场景

> 所属文档：Gate 需求文档 · [返回索引](./README.md)

---

## 2.1 用户角色

| 角色 | 描述 | 主要需求 |
|------|------|----------|
| **普通开发者** | 日常使用各类工具，偶尔切换网络环境 | 快速设置/清除代理、预设切换 |
| **企业开发者** | 在公司内网和家庭网络间频繁切换 | 多预设管理、自动持久化 |
| **AI 开发者** | 大量使用 AI API 和本地 LLM 工具 | 批量配置 AI 工具代理 |
| **Unity 开发者** | 在 Unity 编辑器内管理代理 | GUI 操作，无需命令行 |
| **DevOps / 脚本用户** | 在 CI/CD 流水线或脚本中调用 | `--json` 输出、非交互模式 |
| **工具贡献者** | 为社区贡献新工具的代理配置 | 声明式 JSON 插件、校验工具 |

---

## 2.2 典型使用场景

### 场景 A：新员工首次配置
```bash
gate wizard   # 4步交互向导：输入代理 → 选择工具 → NO_PROXY → 保存预设
```

### 场景 B：在家/公司间切换
```bash
gate preset load home      # 切换到家庭网络
gate preset load office    # 切换到公司内网
```

### 场景 C：临时为某工具配置代理
```bash
gate app git http://127.0.0.1:7890
```

### 场景 D：CI/CD 脚本集成
```bash
gate --json           # JSON 格式输出，便于脚本解析
gate test --json      # 测试结果以 JSON 返回
# 典型 CI 用法：在 pipeline 开始时 gate preset load ci-proxy，结束时 gate clear --all
```

### 场景 E：迁移到新机器
```bash
gate export-all gate-backup.json   # 旧机器导出（含预设、工具路径、全局配置）
gate import-all gate-backup.json   # 新机器恢复
```

### 场景 F：AI 开发批量配置
```bash
gate set http://127.0.0.1:7890 cursor,vscode,ollama,openai,anthropic
gate preset save ai-dev
```

### 场景 G：代理不稳定时自动选最快节点
```bash
gate pick http://p1:7890,http://p2:7890,http://p3:7890 --apply
# 并发测试三个代理，自动选延迟最低的并设置（P1 功能）
```

### 场景 H：长时间使用代理后监测稳定性
```bash
gate watch --interval 60 --notify
# 每 60s 测试当前代理，失败时推送系统通知（P1 功能）
```

### 场景 I：修改工具配置前先备份
```bash
gate backup                        # 备份所有工具配置文件原始内容
gate app --all http://127.0.0.1:7890  # 批量修改
# 若出问题：
gate rollback                      # 一键恢复（P1 功能）
```

### 场景 J：为非标准路径安装的工具配置代理
```bash
# NVM 管理的 Node 安装在 ~/.nvm/versions/node/v20/bin/npm
gate path -n npm --exec ~/.nvm/versions/node/v20/bin/npm
gate path -n npm --config ~/.nvm/versions/node/v20/lib/node_modules/npm/.npmrc
gate app npm http://127.0.0.1:7890
```

### 场景 K：贡献新工具到社区
```bash
# 1. 创建 tool.json
cat > /tmp/mytool/tool.json << 'EOF'
{ "toolName": "mytool", "category": "其他工具",
  "config": { "probePaths": ["~/.mytoolrc"], "format": "keyvalue",
    "proxyLines": [{"key": "proxy", "value": "{proxy}"}] } }
EOF
# 2. 校验
gate plugin validate /tmp/mytool/tool.json
# 3. 本地测试
gate plugin install /tmp/mytool/tool.json
gate app mytool http://127.0.0.1:7890
# 4. 提交 PR 到 gate-community/gate-tools
```
