## P1 功能

### P1.1 代理有效期（`--expires`）

```bash
gate set http://127.0.0.1:7890 --expires 8h
gate set http://127.0.0.1:7890 --expires 1d
gate set http://127.0.0.1:7890 --expires 2026-03-18T09:00:00
```

- `config.json` 新增 `proxyExpiresAt`（ISO 8601 UTC）
- `gate` 无参启动时检查 `proxyExpiresAt`，过期自动执行 `clear`
- 状态总览显示剩余时间；< 1h 时橙色警告
- `--expires` 格式：`Nh`（小时）、`Nd`（天）、ISO 8601 日期时间；过去的时间为参数错误（退出码 1）

### P1.2 PAC 文件生成与托管

```bash
gate pac generate --proxy http://127.0.0.1:7890 --output proxy.pac
gate pac generate --proxy http://127.0.0.1:7890 --bypass localhost,*.local
gate pac serve --port 8889
gate pac serve --port 8889 --file proxy.pac
```

- 生成标准 `FindProxyForURL(url, host)` JS 函数
- NO_PROXY 规则转为 `shExpMatch` 条件
- `serve` 仅监听 `127.0.0.1`（不对外暴露），前台阻塞，Ctrl+C 停止
- 不支持 HTTPS 托管，不支持后台守护模式

### P1.3 社区插件在线安装

```bash
gate plugin search <keyword>        # 搜索社区索引
gate plugin install <id>            # 从社区安装
gate plugin install <id> --force    # 覆盖已安装
gate plugin update <name>           # 更新指定插件
gate plugin update --all            # 更新所有社区插件
```

**索引 JSON 格式**（`index.json`）：
```json
{
  "version": 1,
  "plugins": [
    {
      "id": "clash",
      "toolName": "clash",
      "description": "Clash 代理客户端",
      "version": "1.0.0",
      "sha256": "abc123...",
      "downloadUrl": "https://raw.githubusercontent.com/gate-community/gate-tools/main/plugins/clash/tool.json",
      "verified": true
    }
  ]
}
```

**社区安装流程**：
1. 拉取索引（缓存 1h，超时走缓存）
2. 按 id 查找，未找到 → 退出码 2
3. 下载 `downloadUrl`
4. SHA256 校验，不匹配 → 退出码 1
5. 调用本地安装逻辑
6. 写 `plugin-info.json`：`installedBy: "community"`、`version`、`sha256`

**更新规则**：仅更新 `installedBy: "community"` 的插件；本地安装的插件不可通过 `update` 命令更新。

### P1.4 补全脚本自动安装

```bash

```

| Shell | 写入路径 |
|-------|----------|
| bash | `~/.bash_completion.d/gate` |
| zsh | `~/.zsh/completions/_gate` |
| fish | `~/.config/fish/completions/gate.fish` |
| pwsh | `$PROFILE` 目录下 `gate.ps1` |

### P1.5 配置备份与回滚

```bash
gate backup [--comment "切换前"]
gate backup list
gate rollback [--index N] [--list] [--dry-run]
```

- 备份存 `{DataDir}/backups/{timestamp}/`，保留最近 5 份（超出自动删最旧）
- 备份内容：所有已安装工具的**配置文件原始内容**（非 Gate 自身数据；Gate 数据用 `export-all`）
- `backup-info.json`：时间戳、注释、工具列表、原始路径映射
- 回滚时将备份文件逐一原子写回对应路径
- 回滚前自动创建一次新备份（防误操作）
- `--dry-run`：只输出将被覆盖的文件列表，不写入

### P1.6 环境变量导出脚本

```bash
gate env --export-script bash   # 输出: export HTTP_PROXY=http://...
gate env --export-script pwsh   # 输出: $env:HTTP_PROXY="http://..."
gate env --export-script fish   # 输出: set -x HTTP_PROXY http://...
```

Shell Hook 内部调用此命令实现持久化，也可单独集成到 CI 脚本。

---

### P1.7 全局配置管理命令

```bash
gate config list
gate config get defaultTestUrl
gate config set defaultTestUrl https://api.github.com
gate config set testTimeoutMs 5000
gate config reset
```

- 读写 `{DataDir}/config.json` 中的 `GlobalConfig` 字段
- `list` 输出：键名 | 当前值 | 默认值 | 说明
- 值类型校验：整数字段必须为正整数，布尔字段必须为 `true`/`false`
- 仅支持 `GlobalConfig` 中已定义的键名（不支持任意键值写入）

---

### P1.8 代理连通性定时监测

```bash
gate watch [--interval 60] [--url https://...] [--notify]
```

- 前台运行，Ctrl+C 停止，不支持后台守护模式
- 每次测试输出：时间戳 | 状态（✓/✗）| 响应时间
- 连续失败 3 次时输出醒目警告行
- `--notify`：Windows 用 Toast 通知，macOS 用 `osascript`，Linux 用 `notify-send`（未安装时降级为终端输出）
- 不支持 `--json`（实时流式，不适合单次 JSON 响应）

---

### P1.9 多代理自动选最快节点

```bash
gate pick http://p1,http://p2,http://p3
gate pick --from-history
gate pick http://p1,http://p2 --apply
```

- 并发测试所有候选代理（复用 `ProxyTester.TestMany()` 逻辑）
- 输出排行榜后询问是否应用；`--apply` 跳过确认直接设置
- `--from-history`：从历史记录中选最快（超时代理立即标记不可用）
- 仅支持 HTTP/HTTPS 代理，不支持 SOCKS 延迟测试

---

### P1.10 系统密钥链存储认证密码

- 检测代理 URL 含 `user:pass@` 时，提取密码存入系统密钥链（Windows CredentialManager / macOS Keychain / Linux libsecret）
- 历史记录和配置文件存脱敏版本：`http://***@host:port`
- 实际使用时从密钥链读取并拼接完整 URL

## P2 功能

### P2.1 Unity GUI 代理历史下拉
- 代理地址输入框下方显示最近 10 条历史（数据源：`ProxyHistory.Load()`）
- 点击历史项填充输入框，不自动提交

### P2.2 Unity GUI 插件管理面板
- 新增「插件」侧边导航项
- 已安装插件列表（名称、分类、安装时间、来源、版本）
- 本地安装/社区安装/更新/卸载/校验 按钮

### P2.3 Unity GUI Doctor 面板
- 新增「诊断」侧边导航项
- 自动运行 `gate doctor` 所有检查项并展示结果
- 「重新检查」和「一键修复」按钮

### P2.4 Unity GUI 主题切换
- 深色（当前）/ 浅色（`#f8fafc` 背景，`#1e40af` 主色）
- 设置保存到 `config.json` 中 `uiTheme` 字段

### P2.5 Unity GUI 预设快速切换浮层
- 编辑器顶部菜单「Gate: [当前预设]」下拉按钮
- 一键切换预设，无需打开完整面板

### P2.6 `ILogger` 接口注入
- Gate.Core 引入 `ILogger` 抽象，替换 `ConsoleStyle` 直接输出
- CLI 注入控制台 logger，Unity GUI 注入 `Debug.Log` logger

### P2.7 配置文件 `schemaVersion` 字段
- `profiles/*.json`、`config.json`、`history.json` 添加 `schemaVersion: 1`
- 格式升级时自动迁移旧格式

## P3 功能

### P3.1 GitHub 仓库治理文件

| 文件 | 内容 |
|------|------|
| `CONTRIBUTING.md` | 贡献流程、代码规范、PR checklist、新工具贡献指南 |
| `.github/ISSUE_TEMPLATE/bug.yml` | OS、版本、复现步骤、期望行为 |
| `.github/ISSUE_TEMPLATE/tool-request.yml` | 工具名、官网、配置格式、代理键名 |
| `.github/ISSUE_TEMPLATE/feature.yml` | 功能描述、使用场景、优先级建议 |
| `.github/PULL_REQUEST_TEMPLATE.md` | 变更类型、测试方式、截图（GUI 变更） |

### P3.2 CI/CD 工作流

| 工作流 | 触发条件 | 产物 |
|--------|----------|------|
| `build.yml` | PR 和 push to main | 三平台（win-x64/linux-x64/osx-x64）自包含二进制 |
| `test.yml` | PR | 运行 Gate.Tests，报告覆盖率 |
| `release.yml` | 推送 tag `v*.*.*` | Release + 三平台二进制 + Changelog |
| `validate-plugins.yml` | 社区工具仓库 PR | 校验所有 `tool.json` 格式 |
