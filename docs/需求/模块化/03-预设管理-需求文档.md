## 1. 功能概述

预设管理模块允许用户将当前完整代理配置（全局代理 + 所有已配置工具代理）保存为命名快照（Profile），并一键恢复。适用于"办公室代理 / 家庭代理 / 无代理"等多场景切换。

---

## 2. 数据结构

**存储路径**：`{DataDir}/profiles/{name}.json`

```json
{
  "name": "office",
  "description": "公司代理配置",
  "isDefault": true,
  "createdAt": "2026-03-18T09:00:00Z",
  "updatedAt": "2026-03-18T10:00:00Z",
  "globalProxy": {
    "httpProxy": "http://proxy.company.com:8080",
    "httpsProxy": "http://proxy.company.com:8080",
    "noProxy": "localhost,127.0.0.1,*.company.com"
  },
  "toolProxies": [
    { "toolName": "git",  "proxy": "http://proxy.company.com:8080" },
    { "toolName": "npm",  "proxy": "http://proxy.company.com:8080" }
  ]
}
```

**预设名称规则**：字母、数字、连字符、下划线，长度 1–64，区分大小写。

## 3. 主要功能列表

### 3.1 列出预设

**命令**：`gate preset`

输出：表格列出所有预设（名称、描述、工具数、默认标记、更新时间）。

### 3.2 保存预设

**命令**：`gate preset save <name> [--desc <desc>]`

- 快照当前进程级全局代理（包含 `NO_PROXY`，即完整 `ProxyConfig`）+ 所有 `ReadProxy() != null` 的工具代理（A8：方案A）
- 若同名预设已存在则覆盖（提示覆盖确认，`--force` 跳过确认）
- 写入 `{DataDir}/profiles/{name}.json`

### 3.3 加载预设

**命令**：`gate preset load <name>`

- 恢复全局代理到进程环境变量
- 逐一写入 `toolProxies` 中的工具配置文件
- 若预设中某工具已卸载，输出 **WARN 并列出跳过的工具名**（A7：方案B）
- 单个工具写入失败不中止，最终摘要报告
- 设置 `ActivePreset` = 预设名称

### 3.4 删除预设

**命令**：`gate preset del <name> [--force]`

- 删除 `{DataDir}/profiles/{name}.json`
- 若该预设为默认预设，同时清除默认标记
- `--force` 跳过确认提示

### 3.5 重命名预设

**命令**：`gate preset rename <old> <new>`

- 重命名文件；新名称已存在则返回错误
- 若旧名称为默认预设，将默认标记转移到新名称

### 3.6 设置默认预设

**命令**：`gate preset set-default <name>`

- 在 `config.json` 中写入 `defaultPreset` 字段
- Shell Hook 模式为 `--preset` 时，每次终端启动自动 `gate preset load <name>`

### 3.7 查看预设详情

**命令**：`gate preset show <name>`

- 展示完整 JSON 内容（格式化输出）
- `--diff`：与当前配置对比，高亮差异

### 3.8 导出 / 导入预设

**命令**：`gate preset export <name> [file]`
- 导出到文件（默认 `{name}.gate-preset.json`）

**命令**：`gate preset import <file> [--as <newname>]`
- 从文件导入；`--as` 重命名；目标已存在时提示确认
- 校验 JSON schema：缺少 `name`/`globalProxy` 字段则拒绝并提示

## 4. 技术实现要点

```csharp
// Gate.Core/Managers/ProfileManager.cs
public class ProfileManager {
    public List<Profile> ListAll();                              // 扫描 profiles/ 目录
    public Profile? Get(string name);                             // 读取并反序列化
    public void Save(string name, Profile profile);              // 序列化写入文件
    public void Delete(string name);                             // 删除文件
    public void Rename(string oldName, string newName);          // File.Move
    public bool Exists(string name);                             // File.Exists
    public void ApplyToEnvironment(Profile profile);             // 写环境变量
    public void ApplyToTools(Profile profile, out ApplyResult result); // 写工具配置
}

public record ApplyResult(int Success, int Skipped, int Failed,
    IReadOnlyList<string> FailedTools);
```

## 5. 异常处理

| 异常场景 | 行为 | 退出码 |
|----------|------|--------|
| 预设名称不合规 | 参数错误 | 1 |
| 预设不存在 | 输出错误，列出已有预设名 | 2 |
| 同名预设已存在（save） | 提示确认，`--force` 跳过 | 0 |
| 导入文件 JSON schema 无效 | 拒绝并输出校验错误信息 | 1 |
| profiles/ 目录不可写 | 输出错误 | 4 |
| 工具写入失败（load） | 继续其他工具，摘要报告 | 0 含警告 |

## 6. 验收标准

- [ ] `gate preset save office` 将当前全局代理和所有工具代理快照到文件
- [ ] `gate preset load office` 恢复全局代理和所有工具代理
- [ ] `gate preset del office` 删除文件，若为默认预设则同时清除默认标记
- [ ] `gate preset rename office work` 重命名成功，旧文件不再存在
- [ ] `gate preset set-default office` 写入 `config.json`，Shell Hook 自动加载
- [ ] `gate preset export office`/`import` 导出导入后数据完全一致
- [ ] 导入 JSON schema 无效时返回退出码 1，不写入任何文件

---

## 7. 边界条件

| 场景 | 期望行为 |
|------|----------|
| 预设名含中文 | 参数错误，退出码 1 |
| 预设包含当前已卸载的工具 | 输出 WARN 并列出跳过的工具名，继续处理其余工具（A7） |
| 预设 `toolProxies` 为空数组 | 合法，仅恢复全局代理 |
| `save` 时无全局代理 | `globalProxy` 存空对象 `{}`，合法 |
| 同时 `save` 和 `load` 同一预设 | 正常覆盖后加载（顺序执行） |
