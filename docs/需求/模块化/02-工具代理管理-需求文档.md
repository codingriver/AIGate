# 02 工具代理管理 — 需求文档

> 所属项目：Gate · 模块化文档 v2.0  
> 关联文档：[GUI布局文档](./02-工具代理管理-GUI布局文档.md) | [数据模型](./07-数据模型-需求文档.md)

## 1. 功能概述

工具代理管理模块负责读写各开发工具自身的代理配置文件（如 `~/.gitconfig`、`~/.npmrc`、`~/.docker/config.json` 等）。与全局代理（环境变量）不同，工具代理写入工具配置文件后**始终生效**，不依赖终端会话。

当前支持约 **214 个**工具，通过 JSON 声明式描述或 C# Configurator 实现。

---

## 2. 主要功能列表

### 2.1 查看工具代理

**命令**：`gate app <tool>`

输出内容：
- 工具配置文件路径（含是否存在、是否可读状态）
- 当前代理键值对
- 工具是否已安装

### 2.2 设置工具代理

**命令**：`gate app <tool> <proxy>`

| 场景 | 行为 |
|------|------|
| 配置文件已存在且有代理行 | 原地更新对应行，不新增重复行 |
| 配置文件已存在但无代理行 | 追加代理行到文件末尾（或对应 section 末尾） |
| 配置文件不存在 | 自动创建文件及父目录，写入代理行 |
| 配置文件不可写 | 返回错误码 `4`，输出明确提示 |

### 2.3 批量设置

**命令**：`gate app git,npm,pip http://127.0.0.1:7890`

- 工具名逗号分隔，不含空格
- 逐一处理，单个失败不中止其余
- 最终输出摘要：成功 N 个 / 跳过 M 个 / 失败 K 个

### 2.4 清除工具代理

**命令**：`gate app <tool> --clear`

- 从配置文件移除代理相关行（`RemoveOnClear == true` 的行）
- 其他配置行保持不变
- 配置文件换行符和编码保持原样

### 2.5 全部工具操作

**命令**：`gate app --all <proxy>` / `gate app --all --clear`

- 对 `ToolRegistry.GetAll()` 中**所有工具**（含未安装、含插件）执行操作（A4：方案B）
- 未安装工具写入失败则跳过，在摘要中列出跳过的工具名
- `--except git,docker`：排除指定工具（逗号分隔）
- GUI 批量操作同样对所有工具执行（含未安装），与 CLI 行为一致；未安装工具跳过并在状态栏汇报（B13）

### 2.6 列出所有工具

**命令**：`gate apps [options]`

| 选项 | 行为 |
|------|------|
| `gate apps` | 按分类分组，显示工具名、分类、已安装（✓/✗）、已配置代理（✓/✗） |
| `gate apps --installed` | 仅显示已安装工具 |
| `gate apps --category "AI 编程助手"` | 仅显示指定分类 |
| `gate apps --json` | JSON 格式输出 |

### 2.7 工具路径管理

**命令**：`gate path [options]`

| 选项 | 行为 |
|------|------|
| `gate path -n <tool>` / `gate path --name <tool>` | 显示自定义路径（若有）及自动检测结果；`-n` 是 `--name` 的缩写（B15） |
| `gate path -n git --exec /usr/local/bin/git` | 写入自定义可执行路径 |
| `gate path -n git --config ~/.gitconfig-work` | 写入自定义配置文件路径 |
| `gate path -n git --clear` | 删除整个自定义条目，恢复自动检测；不保留 `null` 空壳（B35） |
| `gate path --list` | 表格显示所有已配置自定义路径的工具 |

存储格式（`{DataDir}/tool-paths.json`）：
```json
{
  "git": { "execPath": "/usr/local/bin/git", "configPath": "~/.gitconfig-work" },
  "npm": { "configPath": "~/.npmrc-company" }
}
```

## 3. 工具安装检测逻辑（详细）

> **C9 三级优先级定义**：① `gate path` 设置的自定义路径（最高）；② `ToolDescriptor.probePaths` 中声明的默认路径（按序尝试）；③ 系统 PATH 中搜索可执行文件（最低）。自定义路径命中则直接返回，不继续往下检测。

```csharp
bool IsInstalled(string toolName) {
    // 1. 自定义可执行路径
    var customExec = _toolPaths.GetExecPath(toolName);
    if (customExec != null) return File.Exists(customExec);

    // 2. 自定义配置文件路径
    var customConfig = _toolPaths.GetConfigPath(toolName);
    if (customConfig != null) return File.Exists(customConfig);

    // 3. 声明的 probePaths（按序尝试）
    foreach (var path in descriptor.Config?.ProbePaths ?? []) {
        if (File.Exists(path.Resolve(currentPlatform))) return true;
    }

    // 4. PATH 中查找可执行文件
    return IsInSystemPath(toolName);
}
```

**优先级**：自定义路径 > probePaths > PATH 检测

## 4. 配置文件写入规范

### 4.1 keyvalue 格式

原地更新规则：扫描文件每行，找到 key 匹配的行则替换；未找到则追加到文件末尾。

```ini
# 原文件
registry=https://registry.npmjs.org
proxy=http://old-proxy:8080

# gate app npm http://127.0.0.1:7890 后
registry=https://registry.npmjs.org
proxy=http://127.0.0.1:7890        # 原地更新
https-proxy=http://127.0.0.1:7890  # 原来没有，追加到末尾
```

### 4.2 ini 格式（含 section）

```ini
# gate app git http://127.0.0.1:7890 后（section="http"）
[core]
repositoryformatversion = 0

[http]
proxy = http://127.0.0.1:7890   # 追加到 [http] section 末尾
                                 # section 不存在时自动创建
```

### 4.2.1 ini section 不存在时的插入规则（B11）

Section 不存在时，追加到文件末尾，格式为：空行 + `[section名]` + 代理行，与主流 git config 格式保持一致：

```ini
# 原文件末尾
[core]
    autocrlf = input

# gate app git http://127.0.0.1:7890 后（[http] section 不存在时）
[core]
    autocrlf = input

[http]
	proxy = http://127.0.0.1:7890
```

### 4.3 json 格式（RFC 7396 Merge Patch）

`ToolConfig.JsonPath` 使用 **JSONPath 格式**（如 `$.proxies.default.httpProxy`），在 `ToolDescriptor` 中补充完整示例（B10）：

```json
// 示例 1：docker（路径：$.proxies.default.httpProxy）
// 示例 2：gradle（路径：$.systemProp.http\.proxyHost）
```

只更新代理相关字段，不触碰其他字段：

```json
// gate app docker http://127.0.0.1:7890 后
{
  "auths": { "registry.example.com": {} },
  "proxies": {
    "default": {
      "httpProxy": "http://127.0.0.1:7890",
      "httpsProxy": "http://127.0.0.1:7890"
    }
  }
}
```

### 4.4 toml 格式

与 ini 格式相同的行扫描逻辑；字符串值需加双引号。

```toml
# gate app cargo http://127.0.0.1:7890 后
[http]
proxy = "http://127.0.0.1:7890"
```

### 4.5 写入安全要求

1. 写入前检查文件可写性；不可写时返回错误，不抛异常
2. 原子操作：先写临时文件（`.tmp` 后缀），成功后 `rename` 替换
3. 文件不存在时自动创建父目录（`Directory.CreateDirectory`）
4. 保留原文件换行符风格（CRLF/LF）
5. 保留原文件编码（UTF-8/GBK 等），**包括 BOM**——读取时检测 BOM，写入时原样保留（A5：方案A）
6. `keyvalue` 格式原地更新时**保留所有注释行**，不删除任何以 `#` 开头的行（A6：方案A）
7. `ClearProxy()` 时删除 `RemoveOnClear == true` 的**所有** key 行（含 `proxyKey` 和 `httpsProxyKey`），不只删除 `proxyKey`（B12）

## 5. 技术实现要点

### 5.1 ToolConfiguratorBase 接口

> **C10 tools/ 目录结构与命名约束**：`tools/` 目录按工具分类组织子目录（如 `tools/ai/cursor.json`、`tools/vcs/git.json`），文件名必须与 `ToolDescriptor.name` 字段完全一致（小写字母 + 连字符，不含空格或大写）。214 为当前最低基准数量，后续可持续新增。

```csharp
public abstract class ToolConfiguratorBase {
    public abstract string ToolName { get; }

    // 将代理写入工具配置文件；文件不可写或不存在时返回 false（不抛异常）
    public abstract bool SetProxy(string proxy);

    // 从配置文件移除代理相关行；返回 false 表示失败
    public abstract bool ClearProxy();

    // 读取当前工具配置文件中的代理值；未配置返回 null
    public abstract string? ReadProxy();

    // 检测工具是否已安装（见上文优先级逻辑）
    public virtual bool IsInstalled() { ... }

    // 获取配置文件路径（用于 gate app <tool> 展示）
    public virtual string? GetConfigFilePath() { ... }
}
```

### 5.2 DeclarativeToolConfigurator（JSON 驱动）

继承 `ToolConfiguratorBase`，通过 `ToolDescriptor` JSON 驱动实现所有方法，无需子类代码。

### 5.3 特殊 C# Configurator 列表

以下工具因配置格式需要程序逻辑处理（命令行调用、JSON merge patch、XML 操作等），无法通过 JSON 声明式驱动，必须使用独立 C# Configurator 实现（C11）：

| 类名 | 工具 | 原因 |
|------|------|------|
| `GitConfigurator` | git | 使用 `git config --global` 命令行写入，不直接操作 `~/.gitconfig` |
| `DockerConfigurator` | docker | JSON 格式需 RFC 7396 merge patch 逻辑，不能简单替换行 |
| `GradleConfigurator` | gradle | 复杂 properties 格式，含变量引用 |
| `MavenConfigurator` | maven | XML `settings.xml` 需 DOM 操作，不能行扫描 |

> **判断规则**：需要调用外部命令、做 JSON/XML 结构化合并、或含复杂变量替换逻辑的工具，使用 C# Configurator；纯 keyvalue / ini / json 字段替换的工具使用 JSON 声明式（`DeclarativeToolConfigurator`）。

### 5.4 工具加载优先级

```
低优先级 ──────────────────────── 高优先级
① EmbeddedResource JSON（tools/**/*.json）
② 特殊 C# Configurator（同名时覆盖 ①）
③ 用户插件（{DataDir}/plugins/{toolName}/tool.json，同名时覆盖 ①②）
```

## 6. 输入输出规则

| 参数 | 约束 |
|------|------|
| 工具名 | 全小写+连字符，不含空格/大写/下划线，长度 ≤ 32 |
| 代理地址 | 同全局代理格式规则（`http(s)://host:port`，端口 1-65535） |
| `--except` | 逗号分隔工具名，不含空格 |
| `--category` | 与 `gate apps` 显示的分类名精确匹配（区分大小写）；分类名允许任意字符串，不限于预定义值（B26）；所有合法分类见 `{DataDir}/支持工具列表.md`（B14） |

## 7. 异常处理

| 异常场景 | 行为 | 退出码 |
|----------|------|--------|
| 工具名不合规 | 输出错误并提示正确格式 | 1 |
| 工具不存在于 ToolRegistry | 输出错误，列出相似工具名 | 2 |
| 工具未安装（检测失败） | 输出警告，跳过（批量时继续） | 批量时 0；单工具时 2 |
| 配置文件不可写 | 输出错误，跳过（批量时继续） | 批量时 0；单工具时 4 |
| XML 格式工具无 C# Configurator | 输出不支持提示 | 1 |

## 8. 验收标准

- [ ] `gate app git http://...` 写入后，`~/.gitconfig` 中出现正确代理行
- [ ] `gate app git --clear` 后，代理行从 `~/.gitconfig` 中消失，其他配置行不变
- [ ] `gate app git,npm,pip http://...` 批量写入，单个失败不中止，最终输出成功/失败数
- [ ] `gate app --all http://...` 对所有已安装工具写入；`--except git` 跳过 git
- [ ] `gate apps` 按分类分组显示，已安装和已配置代理状态正确
- [ ] `gate path -n git --config ~/.gitconfig-work` 后，`gate app git` 使用自定义配置文件
- [ ] 配置文件不存在时自动创建，父目录不存在时自动创建
- [ ] 写入操作不改变原文件换行符和编码
- [ ] 原子写入：中断后不产生半写文件

## 9. 边界条件

| 场景 | 期望行为 |
|------|----------|
| 配置文件为空 | 写入代理行，不报错 |
| 配置文件只有注释 | 保留注释，追加代理行 |
| 同时设置和清除（`--clear` 与代理地址同时传） | 参数错误，退出码 1 |
| 批量操作部分失败 | 继续处理剩余工具，最终摘要列出失败工具 |
| 工具名含数字（如 `pip3`） | 合法（数字允许） |
| `--except` 包含不存在的工具名 | 输出警告，不影响其他工具 |
| probePaths 全部不存在，PATH 也无该工具 | 视为未安装，`gate app --all` 时跳过 |

## 10. 性能指标

| 操作 | 目标 |
|------|------|
| `gate app <tool> <proxy>` | < 100ms（不含文件 I/O 等待） |
| `gate apps`（214 工具加载） | < 200ms |
| `gate app --all <proxy>` | 与工具数和文件 I/O 成正比，单工具 < 20ms |
