## 2. GlobalConfig（config.json）

> **C22 完整字段定义表**（唯一权威来源，各模块文档中提到的 GlobalConfig 字段均以此为准）：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `autoSaveHistory` | bool | `true` | 每次 `gate set` 后是否自动保存到历史记录 |
| `maxHistoryCount` | int | `20` | 历史记录最大条数上限 |
| `defaultTestUrl` | string | `"https://www.google.com"` | 连通性测试默认目标 URL（B21） |
| `defaultTimeout` | int | `10000` | 连通性测试默认超时（毫秒） |
| `defaultPreset` | string | `""` | 默认预设名；空字符串表示无默认预设（B17） |
| `pluginRegistryUrl` | string | `""` | 社区插件索引 URL（C18；P1 阶段填写正式地址） |
| `colorEnabled` | bool | `true` | 是否启用 ANSI 彩色输出 |
| `proxyExpiresAt` | DateTime? | `null` | 代理过期时间（P1；过期后自动清除全局代理并置为 null，A13/B8） |

```json
{
  "autoSaveHistory": true,
  "maxHistoryCount": 20,
  "defaultTestUrl": "https://www.google.com",
  "defaultTimeout": 10000,
  "defaultPreset": "",
  "pluginRegistryUrl": "",
  "colorEnabled": true,
  "proxyExpiresAt": null
}
```

```csharp
public class GlobalConfig {
    public bool   AutoSaveHistory    { get; set; } = true;
    public int    MaxHistoryCount    { get; set; } = 20;
    public string DefaultTestUrl     { get; set; } = "https://www.google.com";
    public int    DefaultTimeout     { get; set; } = 10000;
    public string DefaultPreset      { get; set; } = "";
    public string PluginRegistryUrl  { get; set; } = "";
    public bool   ColorEnabled       { get; set; } = true;
    public DateTime? ProxyExpiresAt  { get; set; } = null; // P1
}
```

**读写规则**：
- 文件不存在时使用默认值，第一次写入时创建
- 文件损坏（非法 JSON）时使用默认值并输出 WARN，不抛异常
- 使用 `System.Text.Json` 序列化，`PropertyNamingPolicy.CamelCase`
- `ProxyExpiresAt`：**仅在 `gate` 无参启动或 `gate info` 时检查**，其他命令执行前不检查（A13：方案A）
- 代理过期自动清除后，`ProxyExpiresAt` 置为 `null` 并写回文件，防止重复触发（B8）

## 3. Profile（profiles/{name}.json）

```json
{
  "name": "office",
  "description": "公司代理",
  "isDefault": true,
  "createdAt": "2026-03-18T09:00:00Z",
  "updatedAt": "2026-03-18T10:00:00Z",

    "noProxy": "localhost,127.0.0.1"

```

```csharp
public class Profile {
    public string   Name        { get; set; } = "";
    public string   Description { get; set; } = "";
    public bool     IsDefault   { get; set; } = false;
    public DateTime CreatedAt   { get; set; }
    public DateTime UpdatedAt   { get; set; }
    public ProxyConfig GlobalProxy { get; set; } = new();
    public List<ToolProxyEntry> ToolProxies { get; set; } = new();
}
// B33：ProxyConfig 改为 class 并提供无参构造函数，保证反序列化时字段缺失不报错
public class ProxyConfig {
    public string? HttpProxy  { get; set; }
    public string? HttpsProxy { get; set; }
    public string? NoProxy    { get; set; }
}
public record ToolProxyEntry(string ToolName, string Proxy);
```

**文件命名**：`{name}.json`，`name` 仅含字母/数字/连字符/下划线。

## 5. ToolDescriptor（EmbeddedResource tools/**/*.json）

> **C23 完整 JSON Schema**：以下为 `ToolDescriptor` 所有字段定义，`tools/` 中每个声明式工具文件必须符合此 Schema。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✓ | 工具唯一标识，小写字母+连字符，与文件名一致 |
| `displayName` | string | | 显示名称，默认同 `name` |
| `category` | string | | 工具分类，允许任意字符串（B26） |
| `description` | string | | 工具简介 |
| `config.format` | string | ✓ | `keyvalue`\|`ini`\|`json`\|`toml` |
| `config.configFiles` | array | ✓ | 至少一个配置文件定义，含 `path` 和 `platform` |
| `config.proxyKey` | string | | keyvalue/ini 格式的代理键名 |
| `config.httpsProxyKey` | string | | HTTPS 代理键名（可选） |
| `config.section` | string | | ini 格式的 section 名称 |
| `config.jsonPath` | string | | json 格式的 JSONPath（如 `$.proxies.default.httpProxy`）（B10） |
| `config.removeOnClear` | bool | | 清除时是否删除代理行，默认 `true` |
| `probePaths` | array | | 安装检测路径列表，含 `path` 和 `platform` |

**`platform` 枚举值**：`all`\|`windows`\|`linux`\|`macos`（B25）

**`PluginInfo` 与 `ToolDescriptor` 区别**：`ToolDescriptor` 描述工具如何配置代理；`PluginInfo` 描述插件的安装元数据。两个文件共存于 `plugins/{name}/` 目录（C24，见第 6 节）。

```json
{
  "name": "npm",
  "displayName": "npm",
  "category": "包管理器",
  "description": "Node.js 包管理器",
  "config": {
    "format": "keyvalue",
    "configFiles": [
      { "path": "~/.npmrc", "platform": "all" }
    ],
    "proxyKey": "proxy",
    "httpsProxyKey": "https-proxy",
    "removeOnClear": true
  },
  "probePaths": [
    { "path": "~/.npmrc", "platform": "all" }
  ]
}
```

```csharp
public class ToolDescriptor {
    public string Name        { get; set; } = "";
    public string DisplayName { get; set; } = "";
    public string Category    { get; set; } = "";
    public string Description { get; set; } = "";
    public ToolConfig Config  { get; set; } = new();
    public List<ProbePath> ProbePaths { get; set; } = new();
}
public class ToolConfig {
    public string Format        { get; set; } = ""; // keyvalue|ini|json|toml
    public List<ConfigFile> ConfigFiles { get; set; } = new();
    public string? ProxyKey     { get; set; }
    public string? HttpsProxyKey { get; set; }
    public string? Section      { get; set; }  // ini 格式的 section 名
    public string? JsonPath     { get; set; }  // json 格式的 JPath
    public bool   RemoveOnClear { get; set; } = true;
}
public record ConfigFile(string Path, string Platform); // platform: all|windows|linux|macos
public record ProbePath(string Path, string Platform);
```

## 6. PluginInfo（plugins/{name}/plugin-info.json）

> **C24 插件目录结构**：插件安装后目录内**两个文件共存**：
> - `tool.json`：工具声明文件，内容等价于内置 `ToolDescriptor`，供 `ToolRegistry` 加载
> - `plugin-info.json`：插件元数据，记录安装时间、来源路径、是否覆盖内置
>
> 目录示例：
> ```
> {DataDir}/plugins/my-tool/
> ├── tool.json          # ToolDescriptor 内容
> └── plugin-info.json   # 插件元数据
> ```

```json
{
  "name": "my-tool",
  "sourcePath": "C:/Users/.../my-tool.json",
  "overridesBuiltin": false,
  "installedAt": "2026-03-18T09:00:00Z"
}
```

```csharp
public record PluginInfo(
    string Name,
    string SourcePath,
    bool OverridesBuiltin,
    DateTime InstalledAt
);
```

## 8. tool-paths.json


## 9. 序列化规范

| 规范项 | 值 |
|--------|----|
| 库 | `System.Text.Json` |
| 命名策略 | `JsonNamingPolicy.CamelCase` |
| 枚举序列化 | 字符串（`JsonStringEnumConverter`） |
| 日期格式 | ISO 8601（`2026-03-18T10:00:00Z`，UTC） |
| null 值 | 默认序列化（保留 null 字段） |
| 格式化 | `WriteIndented = true`（所有持久化文件） |
| 文件编码 | UTF-8 无 BOM |
| 写入方式 | 先写 `.tmp`，再 `File.Replace`（原子操作） |

> **B34 编码区分**：Gate 自身数据文件（`config.json`、`history.json`、`profiles/*.json`、`tool-paths.json` 等）统一 UTF-8 无 BOM；用户工具配置文件（`~/.npmrc`、`~/.gitconfig` 等）读取时检测原文件编码，写入时保留原 BOM（A5）。

## 10. 验收标准

- [ ] `config.json` 损坏时 Gate 使用默认值启动，不崩溃
- [ ] `profile.json` 所有字段均有默认值，缺字段时反序列化不报错
- [ ] `history.json` 超出 MaxHistoryCount 时自动截断最旧条目
- [ ] 所有文件写入使用原子操作（`.tmp` → rename）
- [ ] `ToolDescriptor` 的 `platform` 字段正确过滤当前平台的 configFiles
