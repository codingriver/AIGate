## 七、后续建议

1. **验收标准确认**：各模块需求文档中的「验收标准」清单建议与 PO 逐条确认后勾选。
2. **UXML 控件名对齐**：GUI 布局文档中定义的所有 `name=` 控件名，在实际 UXML 文件创建时须严格一致，建议以本文档为基准做 diff 检查。
3. **原始文档归档**：`docs/需求/`（非模块化）目录建议在各文档顶部统一添加归档说明：
   ```
   > ⚠ 此文档为原始叙述版本，已由 `docs/需求/模块化/` 下对应模块文档取代。仅供背景参考。
   ```
4. **P1 功能占位**：GUI 布局文档中标注「P1 功能」的控件（如备份/回滚按钮、社区插件安装）已设置为禁用状态并附 tooltip，待 P1 开发启动后解除禁用。
5. **McpLogWatcher 实现**：`09-MCP服务器-GUI布局文档.md §7` 提供了完整的 `McpLogWatcher` 和 `GateCliRunner.StartBackground` 实现参考，可直接用于开发。
