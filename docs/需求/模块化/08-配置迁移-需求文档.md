# 08 配置迁移 — 需求文档

> 所属项目：Gate · 模块化文档 v2.0

## 1. 功能概述

配置迁移模块提供 Gate 全量数据的导出/导入能力，以及工具配置文件备份/回滚（P1），支持跨机器迁移和数据保护。

---

## 2. 主要功能

### 2.1 导出全部配置

**命令**：`gate export-all <file>`

导出内容（ZIP 压缩）：
- `config.json`、`history.json`、`tool-paths.json`
- `profiles/*.json`（所有预设）
- `plugins/*/tool.json`（所有插件声明）
- `export-meta.json`（元数据）

`export-meta.json`：
```json
{ "exportedAt": "2026-03-18T10:00:00Z", "gateVersion": "1.2.3",
  "platform": "win32", "schemaVersion": 1 }
```

### 2.2 导入全部配置

**命令**：`gate import-all <file> [--overwrite]`

1. 解压到临时目录
2. 校验 `schemaVersion`（当前仅支持 v1）
3. 逐一复制文件到 DataDir（`--overwrite` 覆盖，否则跳过冲突）
4. **自动**热重载 `ToolRegistry` 和 `ProfileManager`（B37：`ImportAll()` 末尾统一调用，无需用户手动操作）
5. 输出导入摘要（导入 N / 跳过 M / 失败 K）

> **B36 注意事项**：`gate export-all` 导出的是 Gate 数据目录内容，不含工具配置文件（`~/.gitconfig`、`~/.npmrc` 等）本身。恢复到新机器后，需执行 `gate preset load <name>` 重新写入各工具配置文件。

### 2.3 工具配置文件备份（P1）

**命令**：`gate backup [--comment <text>]`

- 将所有已安装且有代理配置的工具配置文件复制到 `{DataDir}/backups/{timestamp}/`
- 写入 `backup-info.json`
- 最多保留 10 个备份（超出自动删最旧）

`backup-info.json`：
```json
{
  "createdAt": "2026-03-18T10:00:00Z",
  "comment": "切换前备份",
  "files": [
    { "toolName": "git", "originalPath": "~/.gitconfig", "backupPath": "git/.gitconfig" }
  ]
}
```

### 2.4 从备份回滚（P1）

**命令**：`gate rollback [--index N] [--list]`

- `--list`：显示所有备份（时间、注释、文件数）
- 无 `--index` 时使用最新备份
- 还原前自动创建新备份（防误操作）

### 2.5 完全重置

**命令**：`gate reset [--force]`

- 删除整个 `{DataDir}` 目录（Gate 自身数据）
- 不删除用户工具配置文件
- `--force` 跳过确认
- 下次任意 Gate 命令启动时**自动重建** DataDir 并写入默认 `config.json`（B38），不报错

## 3. 技术实现

```csharp
// Gate.Core/Managers/MigrationManager.cs
public class MigrationManager {
    public void ExportAll(string outputPath);                          // 打包为 ZIP
    public ImportResult ImportAll(string zipPath, bool overwrite);    // 解压恢复
    public void Reset();                                               // 删除 DataDir
}
public record ImportResult(int Imported, int Skipped, int Failed, string? Error);
```

ZIP 操作使用 `System.IO.Compression.ZipFile`，无第三方依赖。

---

## 4. 异常处理

| 场景 | 行为 | 退出码 |
|------|------|--------|
| 导出路径不可写 | 输出错误 | 4 |
| 导入文件不存在 | 参数错误 | 1 |
| ZIP 格式损坏 | 输出错误，不修改 DataDir | 1 |
| schemaVersion 不支持 | 输出错误，列出支持版本 | 1 |
| 个别文件复制失败 | 记录摘要，继续其余文件 | 0 含警告 |
| `reset` 未 `--force` | 交互确认 | - |

## 5. 验收标准

- [ ] `gate export-all backup.zip` 生成 ZIP，含所有 DataDir 文件
- [ ] `gate import-all backup.zip` 成功恢复所有文件，摘要数量正确
- [ ] `gate import-all` 跳过已有文件（无 `--overwrite`）
- [ ] `gate reset --force` 删除 DataDir，不影响 `~/.gitconfig` 等工具文件
- [ ] 导入 schemaVersion 不支持时退出码 1
