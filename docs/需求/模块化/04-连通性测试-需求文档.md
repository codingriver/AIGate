# 04 连通性测试 — 需求文档

> 所属项目：Gate · 模块化文档 v2.0  
> 关联文档：[GUI布局文档](./04-连通性测试-GUI布局文档.md)

---

## 1. 功能概述

连通性测试模块通过向可达目标发送 HTTP 请求来验证代理是否可用，并测量延迟。支持单代理测试、多代理并发对比、自定义目标 URL 及 JSON 格式输出。

---

## 2. 主要功能

### 2.1 测试当前代理

**命令**：`gate test`

- 读取进程级 `HTTP_PROXY`，若未设置则输出提示并以退出码 `2` 退出（A10：方案A，不测试直连）
- 向 `GlobalConfig.DefaultTestUrl`（默认 `https://www.google.com`，用户可通过 `gate config set` 修改）发送 HEAD 请求（B21）
- 默认 URL 列表按顺序尝试：第一个成功即返回，失败才尝试下一个；`--json` 的 `url` 字段填实际使用的 URL（B20）
- 输出：延迟（ms）、HTTP 状态码、是否成功

### 2.2 测试指定代理

**命令**：`gate test <proxy>`

- 不依赖当前环境变量，直接使用指定代理
- 其余行为同 2.1

### 2.3 自定义测试 URL

**命令**：`gate test [proxy] --url <url>`

- 默认 URL 列表（按顺序尝试）：
  1. `https://www.google.com`
  2. `https://api.github.com`
  3. `https://httpbin.org/get`
- `--url` 指定单个 URL 覆盖默认列表
- URL 必须以 `https://` 或 `http://` 开头，否则退出码 `1`

### 2.4 多代理并发对比

**命令**：`gate test --compare <p1>,<p2>[,<p3>...]`

- 并发测试所有代理，输出对比表格
- 结果按延迟升序排列，最快标记 `★`
- **并发数说明（C14）**：无硬性上限；超过 5 个代理时输出 WARN「对比代理数量较多，可能影响测试准确性」；超时配置对每个代理独立生效

输出示例：
```
代理                            延迟     状态
★ http://127.0.0.1:7890         123ms    200 OK
  http://127.0.0.1:7891         456ms    200 OK
  http://127.0.0.1:7892         超时     ✗
```

### 2.5 超时配置

**命令**：`gate test --timeout <ms>`

- 默认 `10000`（10 秒）
- 范围：100–300000（0.1s–5min），超出范围退出码 `1`

### 2.6 JSON 输出

**命令**：`gate test --json`

```json
{
  "proxy": "http://127.0.0.1:7890",
  "url": "https://www.google.com",
  "success": true,
  "latencyMs": 123,
  "statusCode": 200,
  "errorMessage": null,
  "testedAt": "2026-03-18T10:00:00Z"
}
```

`gate test --compare --json` 输出**数组**，每个元素与上述格式一致，追加 `isBest: bool` 字段标识最快节点（B22）：
```json
[
  { "proxy": "http://127.0.0.1:7890", "latencyMs": 123, "isBest": true, ... },
  { "proxy": "http://127.0.0.1:7891", "latencyMs": 456, "isBest": false, ... }
]
```

### 2.7 代理定时监测（P1）

**命令**：`gate watch [proxy] --interval <sec> [--notify]`

- 每隔 N 秒测试一次，默认 60 秒
- `--notify`：失败时通过**系统原生通知**推送告警（Windows Toast / macOS NSUserNotification）；无 GUI 环境时降级为终端输出 `[WARN]` 行（C15）
- `Ctrl+C` 停止监测

### 2.8 自动选择最快节点（P1）

**命令**：`gate pick <p1>,<p2>[,...] [--apply]`

- 并发测试，选延迟最低且成功的代理
- `--apply`：自动将最快代理 `gate set`；若最快代理与当前代理相同，仍输出提示「当前代理 xxx 已是最快节点，无需切换」，不重复写入历史（C16）

## 3. 技术实现要点

```csharp
// Gate.Core/Services/ProxyTester.cs
public class ProxyTester {
    public ProxyTestResult Test(string proxy, string? url = null, int timeoutMs = 10000);
    public List<ProxyTestResult> TestMany(IEnumerable<string> proxies, string? url = null,
        int timeoutMs = 10000);  // 并发，Task.WhenAll
}

public record ProxyTestResult(
    string Proxy,
    string Url,
    bool Success,
    int? LatencyMs,
    int? StatusCode,
    string? ErrorMessage,
    DateTime TestedAt
);
```

**HTTP 客户端配置**：
```csharp
var handler = new HttpClientHandler {
    Proxy = new WebProxy(proxy),
    UseProxy = true,
    // A9：默认忽略 SSL 证书（兼容内网代理）；证书验证失败时输出 WARN，不中止测试
    ServerCertificateCustomValidationCallback = (msg, cert, chain, errors) => {
        if (errors != SslPolicyErrors.None) {
            Console.Error.WriteLine($"[WARN] SSL 证书验证失败: {errors}，继续测试");
        }
        return true;  // 始终返回 true，不因证书问题中止
    }
};
using var client = new HttpClient(handler) { Timeout = TimeSpan.FromMilliseconds(timeoutMs) };
```

**延迟测量**：使用 `Stopwatch.StartNew()` 包裹 `SendAsync`，取 `Elapsed.TotalMilliseconds`。

## 4. 异常处理

| 异常场景 | 行为 | 退出码 |
|----------|------|--------|
| 代理格式错误 | 参数错误 | 1 |
| URL 格式错误 | 参数错误 | 1 |
| 超时参数越界 | 参数错误 | 1 |
| 当前无代理（`gate test` 无参数） | 提示未设置代理 | 2 |
| 请求超时 | `Success=false`，`ErrorMessage="超时"` | 3 |
| DNS 解析失败 | `Success=false`，`ErrorMessage="DNS解析失败"` | 3 |
| 连接被拒绝 | `Success=false`，`ErrorMessage="连接被拒绝"` | 3 |

---

## 5. 验收标准

- [ ] `gate test http://127.0.0.1:7890` 输出延迟和 HTTP 状态码
- [ ] `gate test --compare p1,p2,p3` 输出对比表格，延迟升序，最快标 ★
- [ ] `gate test --timeout 500` 超时后 `Success=false`，`ErrorMessage` 非空
- [ ] `gate test --json` 输出合法 JSON
- [ ] 代理不可达时退出码为 `3`
