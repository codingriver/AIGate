# 各模块周边可选功能详情表

> 所属项目：Gate · 模块化文档 v2.0  
> 优先级：高（推荐实现）/ 中（预留接口）/ 低（后期拓展）

---

## 模块一：全局代理管理

### F1-1 代理有效期与自动过期（高）
- **场景**：公司设代理后忘清除，回家网络异常。
- **可行性**：低。`GlobalConfig.proxyExpiresAt` + 启动时检查。
- **步骤**：`--expires 8h` 写入过期时间 → 入口 `ExpiryChecker.CheckAndClear()` → GUI 显示剩余时间。
- **关联**：不影响原有逻辑。

### F1-2 代理连通性定时监测（高）
- **场景**：代理不稳定，自动感知断线并通知。
- **可行性**：中。后台定时 + 系统通知 API。
- **步骤**：`gate watch --interval 60 --notify` → 每 N 秒测试 → 失败推送通知。
- **关联**：独立后台进程，依赖 `ProxyTester`。

### F1-3 多代理自动选最快节点（高）
- **场景**：多节点自动选延迟最低。
- **可行性**：低。`Task.WhenAll` 并发测试。
- **步骤**：`gate pick p1,p2 --apply` → 并发测试 → 选最低延迟 → 自动 `gate set`。
- **关联**：依赖 `ProxyTester`、`EnvVarManager`。

### F1-4 系统密钥链存储认证密码（高）
- **场景**：代理含 user:pass，不应明文存 history.json。
- **可行性**：中。Windows CredentialManager / macOS Keychain / Linux libsecret。
- **步骤**：检测 `user:pass@` → 存密钥链 → 历史存脱敏版 → 使用时读取拼接。
- **关联**：影响 `ProxyHistory.Add()` 和 `EnvVarManager.SetProcess()`。

### F1-5 代理历史下拉 GUI（中）
- **场景**：GUI 频繁切换代理，需快速选历史。
- **可行性**：低。输入框 focus 时展示历史 ListView。
- **步骤**：focus → 读 `ProxyHistory.Load()` → 构建下拉 → 点击填入并收起。
- **关联**：仅 GUI 层，不影响 CLI。

## 模块二：工具代理管理

### F2-1 工具代理写入验证（中）
- **场景**：写入后验证是否生效，防止静默失败。
- **可行性**：低。`SetProxy()` 后调 `ReadProxy()` 对比。
- **步骤**：写入后读取验证，不一致输出警告。
- **关联**：`SetProxy()` 末尾追加，不影响其他模块。

### F2-2 工具代理依赖联动（中）
- **场景**：设 pnpm 时自动联动 npm 配置。
- **可行性**：中。`ToolDescriptor` 新增 `dependsOn` 字段。
- **步骤**：`dependsOn: ["npm"]` → 设置时提示用户是否级联。
- **关联**：需变更 `ToolDescriptor` schema。

### F2-3 支持 XML 格式工具（声明式）（中）
- **场景**：Maven `settings.xml` 等 XML 工具无需手写 C# Configurator。
- **可行性**：中。实现 `XmlToolConfigurator`，通过 XPath 定位代理节点。
- **步骤**：`ToolDescriptor` 添加 `xmlPath` 字段 → 实现 XPath 读写逻辑。
- **关联**：新增 Configurator 类，不影响现有 JSON/INI 工具。

## 模块三：预设管理

### F3-1 预设定时切换（中）
- **场景**：工作时间用公司代理，下班自动切换家用。
- **可行性**：中。cron 表达式 + 后台服务。
- **步骤**：`gate preset schedule office "0 9 * * 1-5"` → 后台服务定时 `preset load`。
- **关联**：新增调度模块，不影响预设核心逻辑。

### F3-2 预设差异对比（高）
- **场景**：加载预设前希望看清会改变哪些配置。
- **可行性**：低。对比当前状态与预设内容，高亮差异。
- **步骤**：`gate preset show office --diff` → 读当前状态 → 逐字段对比 → 高亮变更。
- **关联**：只读操作，不影响核心逻辑。

### F3-3 云同步预设（低）
- **场景**：多台机器共享同一套代理预设。
- **可行性**：高难度。需 OAuth 和云存储（GitHub Gist 或私有 API）。
- **步骤**：`gate preset sync --provider gist` → OAuth 认证 → push/pull profiles/。
- **关联**：完全独立模块，不影响本地预设操作。

## 模块四：连通性测试

### F4-1 测试历史记录（中）
- **场景**：查看历史测试结果，了解代理稳定性趋势。
- **可行性**：低。保存最近 50 条到 `{DataDir}/test-history.json`。
- **步骤**：每次测试后追加结果 → `gate test --history` 展示。
- **关联**：在 `ProxyTester.Test()` 后追加保存逻辑。

### F4-2 测试结果 GUI 图表（低）
- **场景**：在 Unity GUI 中以折线图展示历史延迟变化。
- **可行性**：高难度。Unity UIToolkit 无内置图表，需自定义绘制。
- **步骤**：读取测试历史 → 用 `MeshGenerationContext` 绘制折线图。
- **关联**：依赖 F4-1，纯 GUI 层改动。

## 模块五：插件系统

### F5-1 社区插件索引搜索与安装（高）
- **场景**：用户想为小众工具（如 Helix Editor）添加代理支持。
- **可行性**：中。需维护社区索引 JSON（托管于 GitHub Pages 或 CDN）。
- **步骤**：`gate plugin search helix` → 拉取索引 → 模糊匹配 → `gate plugin install helix-editor` → 下载并安装。
- **关联**：依赖 `PluginManager.Install()`，复用本地安装逻辑。

### F5-2 插件版本管理（中）
- **场景**：插件更新后提醒用户升级。
- **可行性**：低。`PluginInfo` 添加 `version` 字段，与索引对比。
- **步骤**：`gate plugin update --all` → 拉取索引 → 对比版本 → 下载新版覆盖安装。
- **关联**：扩展 `PluginManager`，不影响核心逻辑。

### F5-3 插件签名验证（中）
- **场景**：防止恶意插件篡改系统配置文件。
- **可行性**：中。SHA-256 哈希 + GPG 签名验证。
- **步骤**：索引中包含 `sha256` 字段 → 下载后验签 → 不匹配则拒绝安装。
- **关联**：在 `PluginManager.Install()` 前添加验签步骤。

## 模块六：CLI 命令

### F6-1 全局配置管理 `gate config`（高）
- **场景**：用户需要修改默认测试 URL、历史记录上限等配置。
- **可行性**：低。读写 `{DataDir}/config.json`。
- **步骤**：`gate config set maxHistoryCount 50` → 更新 `GlobalConfig` 并序列化。
- **关联**：依赖 `GlobalConfig`，已有数据结构，只需添加命令。

### F6-2 PAC 文件生成与托管（高）
- **场景**：公司网络需要 PAC 文件，用户希望自动生成并托管。
- **可行性**：中。生成 PAC JS 文件 + 内置 HTTP 服务器托管。
- **步骤**：`gate pac generate --proxy http://... --bypass localhost` → 生成 `.pac` 文件 → `gate pac serve --port 8090` 启动 HTTP 服务。
- **关联**：独立模块，不影响现有代理管理逻辑。

### F6-3 补全自动安装（高）
- **场景**：减少用户手动配置 Tab 补全的步骤。
- **可行性**：低。检测当前 shell 并自动写入 profile。
- **步骤**：`gate completion install` → 检测 shell → 写入补全脚本到 profile → 提示重启终端。
- **关联**：复用 `install-shell-hook` 逻辑。

## 模块七：MCP 服务器

### F7-1 MCP 认证（API Key）（高）
- **场景**：SSE 模式对外暴露时，需要防止未授权访问。
- **可行性**：低。Bearer Token 验证（`Authorization: Bearer <key>`）。
- **步骤**：`gate mcp --sse --auth-key mykey` → 启动时记录 key → 每个请求验证 Authorization 头。
- **关联**：仅 SSE 传输层，不影响 stdio 模式和工具逻辑。

### F7-2 工具调用日志（高）
- **场景**：审计 AI 助手对代理的操作记录。
- **可行性**：低。每次 `tools/call` 后追加日志行。
- **步骤**：`DispatchAsync()` 后记录 `{timestamp, tool, args, result}` 到 `{DataDir}/mcp.log`。
- **关联**：在 `McpServer.DispatchAsync()` 添加日志中间件，不影响工具逻辑。

### F7-3 多客户端并发 SSE（高）
- **场景**：Cursor 和 Claude Desktop 同时连接同一个 Gate MCP 服务。
- **可行性**：中。维护 SSE 连接池，广播状态变更事件。
- **步骤**：`SseTransport` 维护 `List<SseClient>` → 状态变更时广播 `notification` 事件。
- **关联**：扩展 `SseTransport`，不影响 stdio 模式。
