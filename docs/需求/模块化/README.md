## 开发使用指南

### 如何查找某功能的需求

1. 查看「功能点列表.md」定位功能所属模块和编号
2. 打开对应模块的「需求文档」查看完整规范
3. 如涉及 GUI，同时打开「GUI布局文档」
4. 如涉及数据结构，参考「07-数据模型-需求文档.md」
5. 如涉及 CLI 命令，参考「06-CLI命令-需求文档.md」

### 如何添加新工具支持

1. 参考「支持工具列表.md」确认工具分类
2. 参考「02-工具代理管理-需求文档.md §5」了解 ToolDescriptor 格式
3. 在 `tools/` 目录对应分类下新建 `{tool-name}.json`
4. 如格式复杂，实现对应 C# Configurator
5. 如需作为插件分发，参考「05-插件系统-需求文档.md」

### 如何处理歧义

1. 打开「歧义点清单.md」
2. 确认每条歧义的修正方案（方案 A 或方案 B）
3. 在清单中标记「已确认」并填写确认方案
4. 按确认方案更新对应模块需求文档
