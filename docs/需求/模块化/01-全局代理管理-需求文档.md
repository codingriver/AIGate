## 1. 功能概述

全局代理管理模块负责管理操作系统环境变量层面的代理配置（`HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY`）。这是 Gate 的核心模块，所有代理设置均从此模块出发，可选同时写入工具配置文件。

**作用层级**：进程级（Process）环境变量，仅对当前终端及其子进程有效。持久化需通过 `gate install-shell-hook` 写入 shell profile。

## 2. 主要功能列表

### 2.1 设置全局代理

**命令**：`gate set <proxy> [tools] [options]`

| 选项 | 行为 |
|------|------|
| `gate set http://127.0.0.1:7890` | 将 `HTTP_PROXY`、`HTTPS_PROXY`、`ALL_PROXY` 同时写入当前进程环境变量（A1：方案B） |
| `gate set http://... git,npm` | 先设置全局代理，再逐一写入指定工具配置文件；工具名逗号分隔，不含空格 |
| `gate set http://... --verify` | 先执行连通性测试（超时 10s），通过后再写入；失败则中止，退出码 `3` |
| `gate set http://... --no-proxy "localhost,127.0.0.1"` | 同时将 `NO_PROXY` 写入进程环境变量 |
| `gate set --history-index 2` | 从 `history.json` 取第 N 条（1-based）地址设置；超出范围退出码 `2` |
| `gate set http://... --expires 8h` | 设置代理过期时间（P1），写入 `config.json` 的 `proxyExpiresAt` 字段 |

**实现要点**：
- 调用 `EnvVarManager.SetProcess("HTTP_PROXY", proxy)`、`EnvVarManager.SetProcess("HTTPS_PROXY", proxy)`、`EnvVarManager.SetProcess("ALL_PROXY", proxy)`（A1：同时写入 ALL_PROXY）
- 若指定 `tools`，通过 `ToolRegistry.Get(toolName)` 逐一调用 `SetProxy(proxy)`
- 单个工具写入失败不中止其余工具，最终摘要列出失败工具名
- 成功写入后调用 `ProxyHistory.Add(proxy)`（当 `GlobalConfig.AutoSaveHistory == true`）
- `--verify` 先调用 `ProxyTester.Test(proxy)`，`Success == false` 时中止，退出码 `3`

### 2.2 清除全局代理

**命令**：`gate clear [options]`

| 选项 | 行为 |
|------|------|
| `gate clear` | 清除进程级 `HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY`、`ALL_PROXY`（B5：与 `gate set` 写入变量对称） |
| `gate clear --all` | 清除全局环境变量（含 `ALL_PROXY`），并对所有工具（含插件）逐一清除代理；**仅清全局代理，不触发过期自动清除**（B7） |

**实现要点**：
- 调用 `EnvVarManager.ClearProcess("HTTP_PROXY")`、`ClearProcess("HTTPS_PROXY")`、`ClearProcess("NO_PROXY")`、`ClearProcess("ALL_PROXY")`（B5）
- `--all` 时遍历 `ToolRegistry.GetAll()`（包含插件工具），对 `ReadProxy() != null` 的工具调用 `ClearProxy()`（A12：包含插件工具）
- 单个工具清除失败记录摘要，不中止
- 代理过期自动清除时：只调用 `gate clear`（不含工具），并将 `config.json` 的 `proxyExpiresAt` 置为 `null`（B7、B8）

### 2.3 查看环境变量三层详情

**命令**：`gate env`

输出格式（表格）：
```
层级       HTTP_PROXY                    HTTPS_PROXY                   NO_PROXY
Machine    (未设置)                      (未设置)                      (未设置)
User       (未设置)                      (未设置)                      (未设置)
Process    http://127.0.0.1:7890         http://127.0.0.1:7890         localhost
生效值     http://127.0.0.1:7890         http://127.0.0.1:7890         localhost
```

- `--write-registry`：将当前代理写入 Windows 注册表；仅 Windows 可用，其他平台退出码 `1`

### 2.4 代理历史记录

**命令**：`gate history` / `gate history clear`

- 每次成功 `gate set` 时自动追加到 `{DataDir}/history.json`
- 最多保留 20 条（由 `GlobalConfig.MaxHistoryCount` 控制）
- 同一地址重复使用时移到最新位置（去重）
- `gate set --history-index N`：1-based，越界退出码 `2`

**history.json 格式**：
```json
[
  { "proxy": "http://127.0.0.1:7890", "usedAt": "2026-03-18T10:00:00Z" }
]
```

### 2.5 Shell 集成（持久化）

**命令**：`gate install-shell-hook [options]`

| 选项 | 行为 |
|------|------|
| `gate install-shell-hook` | 自动检测 shell，写入对应 profile 文件 |
| `--shell bash/zsh/fish/pwsh` | 指定目标 shell |
| `--preset <name>` | Hook 执行 `gate preset load <name>` 而非导出当前代理 |
| `--dry-run` | 只打印要写入的内容和目标文件，不实际修改 |

写入示例（bash/zsh，追加到 `~/.bashrc` 或 `~/.zshrc`）：
```bash
# Gate proxy hook
if command -v gate >/dev/null 2>&1; then
  eval "$(gate env --export-script bash)"
fi
```

> **B4 补充**：`gate env --export-script <shell>` 子命令输出规范：
> - 当前无代理时：输出空脚本（0 行，退出码 0）
> - 有代理时输出三行导出语句：
>   - `bash`/`zsh`：`export HTTP_PROXY=xxx`、`export HTTPS_PROXY=xxx`、`export ALL_PROXY=xxx`
>   - `fish`：`set -x HTTP_PROXY xxx` 等三行
>   - `pwsh`：`$env:HTTP_PROXY = "xxx"` 等三行
> - 此子命令在 CLI 文档（模块六）中补充独立条目

PowerShell（追加到 `$PROFILE`）：
```powershell
# Gate proxy hook
if (Get-Command gate -ErrorAction SilentlyContinue) {
    gate env --export-script pwsh | Invoke-Expression
}
```

fish（写入 `~/.config/fish/conf.d/gate.fish`）：
```fish
# Gate proxy hook
if command -v gate >/dev/null
  gate env --export-script fish | source
end
```

### 2.6 代理有效期（P1）

**命令**：`gate set http://... --expires 8h`

- 支持格式：`Nh`（N 小时）、`Nd`（N 天）、混合单位 `2d12h`/`1h30m`/`30m`、ISO8601 绝对时间（A3：支持混合单位）
- 过期时间写入 `{DataDir}/config.json` 的 `proxyExpiresAt` 字段
- `gate` 无参数启动或 `gate info` 时检查该字段，若过期则自动 `clear`（仅全局代理，不含工具）、将 `proxyExpiresAt` 置 `null` 写回 `config.json`，并输出提示（A13、B7、B8）
- 状态总览显示剩余时间，< 1h 时字体色 `#f59e0b`（警告黄）

## 3. 技术实现要点

### 3.1 核心类与接口

```csharp
// Gate.Core/Managers/EnvVarManager.cs
public class EnvVarManager {
    // 读取当前生效值（Process > User > Machine，取第一个非空）
    public string? GetEffectiveProxy(string key);

    // 写入进程级
    // 内部：Environment.SetEnvironmentVariable(key, value, EnvironmentVariableTarget.Process)
    public void SetProcess(string key, string value);
    public void ClearProcess(string key);  // 传 null 值清除

    // 读取三层详情
    public EnvLayerInfo GetAllLayers(string key);
    // EnvLayerInfo { string? Machine; string? User; string? Process; string? Effective; }

    // 写入 Windows 注册表（非 Windows 平台抛 PlatformNotSupportedException）
    public bool WriteWindowsRegistry(string httpProxy, string httpsProxy, string noProxy);

    // 生成 shell 导出脚本字符串
    public string GenerateExportScript(string shell); // "bash"|"zsh"|"fish"|"pwsh"
}
```

```csharp
// Gate.Core/Managers/ProxyHistory.cs
public class ProxyHistory {
    public ProxyHistory(string historyFilePath);      // 注入路径，便于测试
    public List<HistoryEntry> Load();                 // 文件不存在返回空列表
    public void Add(string proxy);                    // 追加并去重，超限移除最旧
    public HistoryEntry? GetByIndex(int index);       // 1-based，越界返回 null
    public void Clear();                              // 清空并删除文件
}
public record HistoryEntry(string Proxy, DateTime UsedAt);
```

### 3.2 代理地址格式校验

```csharp
public static bool IsValidProxy(string proxy) {
    if (!Uri.TryCreate(proxy, UriKind.Absolute, out var uri)) return false;
    if (uri.Scheme != "http" && uri.Scheme != "https" && uri.Scheme != "socks5") return false;
    if (uri.Port < 1 || uri.Port > 65535) return false;
    if (proxy.EndsWith('/')) return false;  // 不允许尾部斜杠
    return true;
}
```

合法示例：`http://127.0.0.1:7890`、`http://user:pass@host:8080`、`socks5://127.0.0.1:1080`  
非法示例：`127.0.0.1:7890`（无协议）、`http://host:99999`（端口越界）、`http://host:7890/`（尾部斜杠）

### 3.3 Unity GUI 异步要求

所有网络操作（含 `--verify` 测试）必须在后台线程执行，禁止在 Unity 主线程调用网络 I/O：

```csharp
// GlobalPanelController 中应用设置按钮回调示例
private async void OnApplyClicked(ClickEvent evt) {
    applyBtn.SetEnabled(false);
    applyBtn.text = "设置中...";
    try {
        if (_testToggle.value) {
            var result = await Task.Run(() => _proxyTester.Test(_proxyInput.value));
            if (!result.Success) { ShowFeedback(result.ErrorMessage, false); return; }
        }
        await Task.Run(() => {
            _envVarManager.SetProcess("HTTP_PROXY",  _proxyInput.value);
            _envVarManager.SetProcess("HTTPS_PROXY", _proxyInput.value);
        });
        ShowFeedback("代理已设置", true);
        Refresh();
    } finally {
        applyBtn.SetEnabled(true);
        applyBtn.text = "应用设置";
    }
}
```

### 3.4 事件注销要求

Unity GUI 中所有 `RegisterCallback` 必须在 `Dispose()` 中对应 `UnregisterCallback`，防止热重载时内存泄漏。

## 4. 输入输出规则

| 参数 | 类型 | 约束 | 错误码 |
|------|------|------|--------|
| `<proxy>` | string | `http(s)://host:port`，端口 1-65535，不含尾部斜杠 | 1 |
| `[tools]` | string | 逗号分隔工具名，须在 ToolRegistry 中存在 | 2 |
| `--no-proxy` | string | 逗号分隔主机/IP/通配符，不做内容校验 | - |
| `--history-index N` | int | 正整数，1-based，须 ≤ 历史条数 | 2 |
| `--expires` | string | `Nh`、`Nd`、混合单位（`2d12h`/`1h30m`）或 ISO8601 日期时间（A3） | 1 |
| `--timeout` | int | 正整数（毫秒），默认 10000 | 1 |

**`gate --json` 输出结构**：
```json
{
  "globalProxy": {
    "httpProxy": "http://127.0.0.1:7890",
    "httpsProxy": "http://127.0.0.1:7890",
    "noProxy": "localhost,127.0.0.1"
  },
  "configuredTools": [
    { "toolName": "git", "category": "版本控制", "proxy": "http://127.0.0.1:7890" }
  ],
  "presets": [ { "name": "office", "isDefault": true, "toolCount": 5 } ],
  "activePreset": null
}
```

## 5. 异常处理

| 异常场景 | 行为 | 退出码 |
|----------|------|--------|
| 代理地址格式不合法 | 输出格式示例，不写入 | 1 |
| `--verify` 测试失败 | 输出失败原因，不写入代理 | 3 |
| 历史索引超出范围 | 输出错误提示 | 2 |
| 环境变量写入失败（权限） | 输出错误，中止 | 4 |
| 工具配置文件写入失败 | 记录在摘要中，继续其他工具 | 0 含警告 |
| `--write-registry` 在非 Windows 平台 | 输出错误 | 1 |
| Shell Hook 目标文件不可写 | 输出错误 | 4 |

## 6. 验收标准

- [ ] `gate set http://127.0.0.1:7890` 后，当前进程 `HTTP_PROXY`、`HTTPS_PROXY`、`ALL_PROXY` 均为该值（A1）
- [ ] `gate clear` 后，三个环境变量均被清除
- [ ] `gate env` 正确展示 Machine/User/Process 三层及生效值
- [ ] `gate set http://... git,npm` 同时更新 git 和 npm 配置文件，单个失败不影响另一个
- [ ] `gate set http://... --verify` 在代理不可达时退出码为 `3`，不写入任何配置
- [ ] `gate history` 显示最近 20 条历史，相同地址不重复出现；输出格式为表格（列：序号、代理地址、最后使用时间 `MM-dd HH:mm`）；`--json` 时输出数组（B6）
- [ ] `gate install-shell-hook` 正确追加 hook 脚本，`--dry-run` 只打印不写入
- [ ] `gate env --write-registry` 在 Windows 写入注册表，非 Windows 返回退出码 `1`
- [ ] `--json` 模式所有输出均为合法 JSON，不含 ANSI 转义码
- [ ] Unity GUI 代理测试在后台线程执行，主线程不阻塞

## 7. 边界条件

| 场景 | 期望行为 |
|------|----------|
| 端口为 0 或 65536 | 参数错误，退出码 1 |
| 代理含尾部斜杠 | 参数错误，退出码 1 |
| `--history-index 0` 或负数 | 参数错误，退出码 1 |
| 工具名不存在 | 输出警告，跳过该工具，不中止 |
| 多终端同时 `gate set` | 以最后写入为准（OS 行为） |
| `gate clear --all` 时工具配置已删除 | 跳过（`ReadProxy()` 返回 null） |
| history.json 损坏 | 视为空历史，覆盖重写 |
| config.json 损坏 | 使用默认值，输出 WARN |
| `--expires` 时间已过去 | 参数错误，退出码 1 |

| 功能 | 优先级 | 说明 |
