|---|----------|----------------|----------------|-----------|----------|
| B40 | MCP 服务器 | `set_tool_proxy` 中 `proxy` 与 `clear: true` 同时传入时，服务器行为未定义 | **返回 `-32602` 参数错误**，`message` 为 `"proxy and clear are mutually exclusive"`；在 JSON Schema 中补充互斥约束说明 | `clear` 优先，忽略 `proxy` 字段 | 方案 A|
| B41 | MCP 服务器 | `get_status` 返回 `configuredTools` 时，若实时遍历 214 个工具读配置文件，每次调用性能开销大 | **只返回 `ReadProxy() != null` 的工具**，并在实现中加简单内存缓存（工具代理写入/清除时更新缓存），避免每次全量读文件 | 每次实时读取全部工具配置文件，不缓存 方案 A| |
| B42 | MCP 服务器 | stdio 模式进程生命周期：是由 Cursor 启动并管理（Cursor 退出时自动终止），还是用户手动启动后 Cursor 连接？ | **由 Cursor 启动并管理**（通过 `mcp.json` 配置的 `command` 字段）；Cursor 退出时自动发送 SIGTERM；在客户端配置示例中补充说明 | 用户手动运行 `gate mcp`，Cursor 连接到已有进程 | 方案 A|
| B43 | MCP 服务器 | SSE 模式 `/messages` POST 端点的 `Content-Type` 要求及非法请求的响应行为未定义 | **必须为 `application/json`**；否则返回 HTTP 400，Body 为 `{"error":"Content-Type must be application/json"}`；在 SSE 端点文档中补充说明 | 宽松处理，任何 Content-Type 都尝试解析 JSON |方案 A |
| B44 | MCP 服务器 | 日志轮转触发时机未定义：每次写日志前检查文件大小，还是定时检查？写日志时轮转是否会丢行？ | **每次写日志前检查文件大小**；超过 10MB 则先轮转再写入，保证当前行不丢失；轮转在同一个写操作的锁内完成 | 定时检查（每分钟），可能丢失轮转时间窗口内的日志行 | 方案 A|
| B45 | MCP 服务器 | GUI 启动 MCP 后若进程意外崩溃，状态卡仍显示"● 运行中"，UI 状态与实际不符 | **定时检查进程存活**（每 5 秒 `_mcpProcess.HasExited`），发现崩溃后自动切换为"已停止"状态并在日志区追加"[ERROR] MCP 进程意外退出" | 不检测，依赖用户感知并手动点"停止" | 方案 A|
| B46 | MCP 服务器 | `McpLogWatcher` 使用 `FileSystemWatcher`；若 `DataDir` 恰好在 Unity 项目 `Assets` 目录内，会与 `AssetDatabase` 文件监听冲突，导致频繁导入触发 | **在文档中明确要求 `DataDir` 必须在 Unity 项目目录外**（默认路径 `AppData\Local\Gate` 已满足此要求）；同时在 GUI 规范文档中加注意事项 | `DataDir` 允许自定义，但若在项目内需将路径加入 `.gitignore` 和 `AssetDatabase` 忽略列表 |方案 A |
| B47 | MCP 服务器 | `GateCliRunner.StartBackground` 中 `RedirectStandardInput = true` 但从不写入任何内容；若 GUI 启动的是 **stdio 模式**，`gate mcp` 进程读 stdin 将永久阻塞，进程挂死 | **GUI 的"启动 MCP"按钮只支持 SSE 模式**；若用户选择 stdio 模式，禁用启动按钮并显示说明文字："stdio 模式由 Cursor / Claude Desktop 自动管理，无需在此手动启动" | 保持现状，允许 GUI 启动 stdio 模式，由用户自行处理 stdin 管道问题 | 方案 A|
| B48 | MCP 服务器 | GUI MCP 面板的 RadioButton 提供 stdio / SSE 两种选项，但 stdio 模式由 AI 客户端（Cursor 等）负责启动和管理，GUI 手动启动 stdio 进程没有实际意义，会造成用户困惑 | **stdio 模式行仅用于展示客户端配置片段**（显示 `mcp.json` 配置示例），启动/停止按钮在 stdio 模式下隐藏；SSE 模式才显示启动/停止按钮 | 保留两种模式均可启动的设计，在 tooltip 中说明 stdio 模式的用途 |方案 A |
