## 5.2 工具加载优先级

```
低优先级 ────────────────────────────── 高优先级

① EmbeddedResource
   Gate.Core/tools/**/*.json
   随 DLL 打包，无外部依赖

② 特殊 C# Configurator
   同名工具时覆盖 ①
   用于格式特殊的工具（XML/复杂TOML等）

③ 用户插件
   {DataDir}/plugins/{toolName}/tool.json
   同名时覆盖 ① 和 ②
   gate plugin install 安装后热注册，无需重启
```

**同名规则**：后加载的覆盖先加载的；覆盖时输出 DEBUG 级日志。

## 5.4 代理层级与读取顺序

```
优先级（高→低）：Process > User > Machine

读取时：优先取 Process 级，若为空则取 User 级，再空则取 Machine 级
写入时：gate set 默认写入 Process 级（当前终端）
        gate env --write-registry 写入 User 级（Windows 注册表）
```

`gate env` 展示三层完整值及最终生效值，便于调试代理冲突问题。

## 5.5 关键设计决策

| 决策 | 原因 | 替代方案及放弃原因 |
|------|------|------------------|
| Gate.Core 用 netstandard2.0 | 兼容 Unity 2021+，DLL 嵌入编辑器 | net8.0：Unity 不兼容 |
| 工具优先 JSON 声明 | 无需编译，社区贡献门槛低 | 纯 C#：每个工具都需 PR+编译 |
| EmbeddedResource 嵌入 JSON | 单文件发布，无外部文件依赖 | 外部文件：部署时需保持目录结构 |
| GatePaths 统一管理存储路径 | 修复 Linux XDG 路径问题 | 路径散落各处：难以测试和跨平台 |
| OutputSettings 全局单例 | 一处控制所有输出模式 | 每个命令各自判断：代码重复 |
| Process 级环境变量 | 不污染系统，安全可控 | User 级：影响所有新终端，难以撤销 |

## 5.6 EnvVarManager 行为规范

```csharp
// 读取（按优先级）
string? GetEffectiveProxy()  // Process > User > Machine，取第一个非空值

// 写入（Process 级，当前进程）
void SetProcess(string key, string value)  // 立即生效，仅限当前进程及子进程

// 写入（User 级，Windows 注册表 / Linux export 到 ~/.profile）
void SetUser(string key, string value)     // 需要管理员权限（Windows Machine级）

// 读取三层详情（供 gate env 展示）
EnvLayerInfo GetAllLayers(string key)  // 返回 { machine, user, process, effective }
```

**注意**：Process 级环境变量通过 `Environment.SetEnvironmentVariable(key, value, EnvironmentVariableTarget.Process)` 设置，**仅对当前进程及其子进程有效**，不影响其他终端。

## 5.7 ToolConfiguratorBase 接口规范

```csharp
public abstract class ToolConfiguratorBase {
    // 必须实现
    public abstract string ToolName { get; }

    // 将代理写入工具配置文件
    // proxy: 完整代理地址，如 http://127.0.0.1:7890
    // 返回：成功 true；配置文件不可写或不存在时返回 false（不抛异常）
    public abstract bool SetProxy(string proxy);

    // 从工具配置文件移除代理相关行
    // 返回：成功 true；配置文件不可写时返回 false
    public abstract bool ClearProxy();

    // 读取当前工具配置文件中的代理值
    // 返回：代理地址字符串；未配置时返回 null
    public abstract string? ReadProxy();

    // 检测工具是否已安装（见 07 §7.5 详细逻辑）
    public virtual bool IsInstalled() { ... }

    // 获取配置文件路径（用于 gate app <tool> 展示）
    public virtual string? GetConfigFilePath() { ... }
}
```

**DeclarativeToolConfigurator** 继承此基类，通过 `ToolDescriptor` JSON 驱动实现所有方法，无需子类。

**特殊 C# Configurator**（如 `GitConfigurator`）继承此基类并重写 `SetProxy`/`ClearProxy`/`ReadProxy`。

## 5.8 GlobalConfigManager 行为规范（P1）

```csharp
public class GlobalConfigManager {
    // 读取 config.json，字段缺失时使用默认值（不报错）
    public GlobalConfig Load();

    // 写入 config.json（原子操作）
    public void Save(GlobalConfig config);

    // 读取单个配置项（供 gate config get 使用）
    public string? Get(string key);

    // 写入单个配置项，校验类型和合法值后写入
    // 不合法时抛 ArgumentException，调用方输出错误
    public void Set(string key, string value);

    // 恢复所有配置项为默认值
    public void Reset();
}
```

**默认值**（`Load()` 时字段缺失的回退值）：

| 字段 | 默认值 |
|------|--------|
| `DefaultPreset` | `null`（不自动加载预设） |
| `DefaultTestUrl` | `"https://www.google.com"` |
| `TestTimeoutMs` | `10000` |
| `AutoSaveHistory` | `true` |
| `MaxHistoryCount` | `20` |
| `ProxyExpiresAt` | `null`（不过期） |
| `UiTheme` | `"dark"` |
