## 10.1 跨平台兼容性

| 平台 | 最低版本 | 特殊说明 |
|------|----------|----------|
| Windows | 10 1903+ | 支持 `%LOCALAPPDATA%`；`--write-registry` 写入 `HKCU\Internet Settings` |
| Linux | Ubuntu 20.04+ / 其他主流发行版 | 遵循 XDG Base Directory（`$XDG_DATA_HOME/gate`） |
| macOS | 12 Monterey+ | 使用 `~/Library/Application Support/gate` |

**三平台行为一致要求**：相同命令在三个平台产生相同的逻辑结果，仅文件路径和系统 API 调用可以有差异。

**路径分隔符**：Gate.Core 内部统一使用 `Path.Combine()` 和 `Path.DirectorySeparatorChar`，不硬编码 `/` 或 `\`。

## 10.3 可靠性

### 文件操作安全

- 所有文件 I/O 有 try/catch，失败时输出明确错误信息，不崩溃
- 写入使用原子操作：先写临时文件（`.tmp` 后缀），成功后 rename 替换
- 单个工具代理写入失败**不影响**其他工具的操作，继续处理后续工具
- `preset load` 的工具代理写入失败时跳过并在摘要中记录，不回滚已成功的工具

### 并发安全

- Gate 不提供跨进程锁，多终端同时运行 `gate set` 可能导致环境变量竞争（这是预期行为，由操作系统处理）
- 文件写入有原子性保障（rename），不会产生半写文件

### 数据完整性

- `export-all` 导出文件包含 `version` 字段，未来格式变更时可检测并提示
- 预设文件中有 `updatedAt` 时间戳，可用于检测文件是否被外部修改

## 10.4 安全性

| 类别 | 要求 | 当前状态 |
|------|------|----------|
| 代理认证密码 | 不以额外方式存储，直接嵌入 URL（`http://user:pass@host`） | 已实现（明文，见约束 12.6） |

| 插件安全 | 安装前格式校验，禁止 DLL 插件 | 已实现 |
| 社区插件校验 | SHA256 校验 + verified 标记 | P1 规划 |
| 非 root 运行 | 不要求提权，用户级操作 | 已实现 |
| 文件权限 | 不修改配置文件的现有权限位 | 已实现 |

## 10.6 可测试性

### 测试范围

| 测试类 | 覆盖内容 |
|--------|----------|
| `ConfigValidatorTests` | 代理地址格式校验（合法/非法地址各 10+ 用例） |
| `ProfileManagerTests` | 保存/加载/删除/重命名/设默认 完整 CRUD 循环 |
| `ProxyHistoryTests` | 追加/去重/上限/清除 |
| `PluginManagerTests` | 安装/校验失败/卸载/覆盖规则 |
| `DeclarativeConfiguratorTests` | keyvalue/ini/json/toml 格式读写，`{proxy}` 模板替换，section 创建 |
| `EnvVarManagerTests` | Process 级设置和清除（不依赖系统注册表的 mock） |
| `GlobalConfigManagerTests`（P1） | Load 默认值回退、Set 类型校验、Reset 恢复默认 |
| `BackupManagerTests`（P1） | 备份创建、备份列表、回滚写回、超出5份自动删旧 |
| `ConfigMigrationTests` | export-all 含 globalConfig/plugins 字段、import-all 同名跳过/覆盖逻辑 |

### 测试原则

- 所有 Manager 接受 `GatePaths` 依赖注入，测试时传入临时目录（`Path.GetTempPath()`）
- 不测试外部网络（`ProxyTester` 在测试中用 mock HTTP client）
- CLI 命令通过 `System.CommandLine` 的 `ParseResult` 单元测试，不启动完整进程
