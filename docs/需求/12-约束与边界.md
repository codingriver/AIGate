# 12 已知约束与边界

> 所属文档：Gate 需求文档 · [返回索引](./README.md)

---

## 12.1 代理协议限制

| 约束 | 说明 | 影响范围 |
|------|------|----------|
| 工具配置文件代理仅支持 HTTP/HTTPS | 写入工具配置文件的代理不支持 `socks4://`、`socks5://` | `gate app`、`preset load` 工具代理部分 |
| SOCKS 代理仅在环境变量层支持 | `HTTP_PROXY=socks5://...` 部分工具会识别，但 Gate 不写入工具配置文件 | `gate set` 全局代理 |
| Gate 不验证工具对 SOCKS 的实际支持 | 是否有效取决于工具本身 | 用户责任 |

---

## 12.2 持久化限制

| 约束 | 说明 | 解决方案 |
|------|------|----------|
| `gate set` 仅写入进程级（当前终端） | 新开终端不生效 | 运行 `gate install-shell-hook` |
| Shell Hook 需重启终端才生效 | 修改 profile 文件后，需新开终端或 `source ~/.zshrc` | 命令完成后提示用户 |
| `gate clear` 不清除 Shell Hook 中的代理 | 若 hook 绑定了预设，下次开终端会重新加载 | `gate install-shell-hook --preset ""` 移除绑定 |

---

## 12.3 平台限制

| 约束 | 说明 |
|------|------|
| `--write-registry` 仅 Windows | Linux/macOS 上调用时输出错误，退出码 `1` |
| Linux/macOS 无系统代理注册表 | 只能通过环境变量影响工具，无法写入系统级 Internet 设置 |
| WSL 环境 | Gate 在 WSL 中以 Linux 版本运行，Windows 宿主机代理不自动同步 |
| macOS SIP | Gate 不修改 `/etc/` 下的系统配置 |

---

## 12.4 插件限制

| 约束 | 说明 |
|------|------|
| 当前仅支持本地文件安装 | `gate plugin install` 只接受本地路径，在线安装在 P1 实现 |
| 仅支持声明式 JSON 格式 | 不支持 DLL/脚本插件（安全考量） |
| 插件不支持版本管理 | 无 `gate plugin update` 命令，在线安装实现后才支持（P1） |
| 插件覆盖内置工具无警告 | 目前只有 DEBUG 日志，未来应有 `[WARN]` 提示 |

---

## 12.5 技术限制

| 约束 | 说明 | 计划 |
|------|------|------|
| Gate.Core 无统一日志框架 | 使用 `ConsoleStyle` 直接输出，无结构化日志 | P2 引入 `ILogger` 接口 |
| 无配置文件 schema 版本字段 | `profiles/`、`config.json` 无版本号，格式升级时需手动处理 | P2 添加 `schemaVersion` 字段 |
| Unity GUI 不支持插件热加载 | 安装插件后需在 Editor 中手动点「刷新」 | P2 实现文件监听自动刷新 |
| 无跨进程锁 | 多终端同时写同一工具配置文件可能冲突 | 用户责任，文档说明 |
| `System.Text.Json` 不支持注释 | `config.json` 等文件不能有 JSON 注释 | 预期行为 |

---

## 12.6 安全限制

| 约束 | 说明 | 计划 |
|------|------|------|
| 代理认证密码明文嵌入 URL | `http://user:pass@host:port` 中密码可见于配置文件和历史记录 | P1 系统密钥链集成 |
| 历史记录含认证信息 | `history.json` 中保存完整代理 URL（含密码） | P1 脱敏存储（只存 `host:port`，密码存密钥链） |
| 无审计日志 | 无法追溯谁在何时修改了代理配置 | P2 可选 `audit.log` |

---

## 12.7 使用边界说明

以下场景超出 Gate 的设计范围，**不支持**且**不计划支持**：

- 管理系统级防火墙规则
- 透明代理（TProxy）配置
- 代理协议转换（如 SOCKS5 转 HTTP）
- 工具的其他网络设置（如 SSL 证书、DNS 配置）
- 多用户/多租户代理隔离
- 容器内部的代理配置（容器外设置后通过环境变量注入）

## 12.8 新增命令（P1）的边界

| 命令 | 边界说明 |
|------|----------|
| `gate watch` | 前台阻塞运行，不支持后台守护进程模式（daemon）；关闭终端即停止监测 |
| `gate watch` | 不支持 `--json` 输出（实时流式监测，不适合单次 JSON 响应模式） |
| `gate watch --notify` | Linux 依赖 `notify-send`（需 `libnotify`），未安装时降级为终端输出，不报错 |
| `gate pick` | 仅测试 HTTP/HTTPS 代理；不支持 SOCKS 代理延迟测试 |
| `gate pick` | `--from-history` 不测试已过期代理（超时立即标记为不可用）|
| `gate pac serve` | 仅支持本地回环地址监听（`127.0.0.1`），不对外暴露；不支持 HTTPS 托管 |
| `gate pac serve` | 前台阻塞运行，Ctrl+C 停止服务；不支持后台运行 |
| `gate backup` | 仅备份工具**配置文件**内容，不备份工具本身的安装包或可执行文件 |
| `gate backup` | 不备份 Gate 自身数据（预设、历史记录等），这些由 `gate export-all` 负责 |
| `gate rollback` | 回滚后需手动重新加载工具（部分工具在运行时读取配置文件，回滚不影响正在运行的进程） |
| `gate rollback --dry-run` | 只输出将被覆盖的文件列表，不实际写入 |
| `gate config set` | 仅支持 `GlobalConfig` 中已定义的键名，不支持任意键值写入 |
| `gate plugin search` | 需要网络访问社区索引；离线时输出错误，退出码 `1` |
| `gate plugin update` | 仅更新 `installedBy: "community"` 的插件；本地安装的插件不可通过此命令更新 |
