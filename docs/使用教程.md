# Gate 使用教程

> 跨平台命令行代理配置管理工具 — 完整使用手册

---

## 安装与快速开始

```powershell
# Windows 构建
.\build.ps1

# Linux / macOS 构建
chmod +x build.sh && ./build.sh
```

产物位于 `publish/gate-{platform}/gate`（或 `gate.exe`）。

```bash
gate wizard    # 交互式向导，4 步完成首次配置（推荐）
gate           # 查看状态总览
gate -h        # 帮助
```

---

## 核心概念

### 代理层级

| 优先级 | 层级 | 生效范围 |
|--------|------|----------|
| 1（最高） | Process（进程级） | 仅当前终端及子进程 |
| 2 | User（用户级，持久化） | 当前用户所有新进程 |
| 3 | Machine（系统级） | 所有用户 |
| 独立 | 工具配置文件 | 该工具始终生效 |

`gate set` 默认写入进程级。运行 `gate install-shell-hook` 实现持久化。

## 命令参考

### gate — 状态总览

```bash
gate           # 全局代理 + 已配置工具 + 预设列表
gate --json    # JSON 格式
```

### gate set

```bash
gate set http://127.0.0.1:7890
gate set http://127.0.0.1:7890 git,npm        # 同时写入工具配置
gate set http://127.0.0.1:7890 --verify       # 设置前测试
gate set http://... --no-proxy "localhost,127.0.0.1"
gate set --history-index 2                    # 从历史选择
```

### gate clear

```bash
gate clear              # 清除全局代理
gate clear git,npm      # 清除工具代理
gate clear --all        # 全部清除
```

### gate app

```bash
gate app git                           # 查看配置
gate app git http://127.0.0.1:7890     # 设置
gate app git,npm,pip http://...        # 批量
gate app git --clear                   # 清除
gate app --all http://...              # 所有已安装工具
gate app --all --except git http://... # 排除
```

### gate apps

```bash
gate apps                  # 所有工具（含安装/代理状态）
gate apps --installed      # 只显示已安装
```

### gate env

```bash
gate env                       # Machine/User/Process 三层明细
gate env --write-registry      # 写入 Windows 系统代理
```

### gate preset

```bash
gate preset                           # 列出所有预设
gate preset save office               # 保存当前配置
gate preset load office               # 加载（恢复全局+工具配置文件）
gate preset del office
gate preset rename office work
gate preset set-default office
gate preset export office backup.json
gate preset import backup.json
```

> `preset load` 同时恢复全局环境变量代理和工具配置文件代理。

### gate test

```bash
gate test                                      # 测试当前代理
gate test http://127.0.0.1:7890               # 测试指定代理
gate test http://... --url https://github.com  # 自定义目标
gate test --compare http://p1,http://p2        # 多代理对比
```

### gate wizard

四步引导：①输入代理地址 ②选择工具（编号菜单）③设置全局代理 ④保存预设

### gate doctor

```bash
gate doctor            # 检查目录权限、配置可写性、持久化、shell hook
gate doctor --verbose  # 显示全部检查项
```

### gate history

```bash
gate history          # 最近 20 条
gate history clear
```

### gate path

```bash
gate path -n git --exec /usr/local/bin/git  # 指定可执行路径
gate path -n git --config ~/.gitconfig-work # 指定配置文件
gate path -n git --clear                    # 恢复自动检测
gate path --list
```

### gate plugin

```bash
gate plugin list
gate plugin install path/to/tool.json
gate plugin remove mytool
gate plugin validate path/to/tool.json
```

### gate export-all / import-all

```bash
gate export-all backup.json
gate import-all backup.json [--overwrite]
```

### gate install-shell-hook

```bash
gate install-shell-hook                   # 自动检测 shell
gate install-shell-hook --shell zsh
gate install-shell-hook --preset office   # 启动时加载预设
gate install-shell-hook --dry-run
```

### gate completion

```bash
gate completion bash > ~/.bash_completion.d/gate
gate completion zsh  > ~/.zsh/completions/_gate
gate completion fish > ~/.config/fish/completions/gate.fish
gate completion pwsh
```

### gate reset

```bash
gate reset          # 交互确认后清除所有配置
gate reset --force  # 跳过确认
```

## 典型工作流

### A：新机器配置
```bash
gate wizard
gate install-shell-hook
gate doctor
```

### B：多网络环境切换
```bash
gate set http://proxy.company.com:8080 --verify
gate preset save office
gate preset load home      # 回家后切换
```

### C：AI 开发批量配置
```bash
gate set http://127.0.0.1:7890 cursor,vscode,ollama,openai,anthropic
gate preset save ai-dev
```

### D：机器迁移
```bash
# 旧机器
gate export-all ~/gate-backup.json
# 新机器
gate import-all ~/gate-backup.json
gate install-shell-hook
```

子目录：`profiles/`、`plugins/`、`history.json`、`tool-paths.json`

| 旧命令 | 新命令 |
|--------|--------|
| `gate global -p http://...` | `gate set http://...` |
| `gate tool -n git -p http://...` | `gate app git http://...` |
| `gate profile --name x --load` | `gate preset load x` |
| `gate apply x` | `gate preset load x` |
| `gate check -p http://...` | `gate test http://...` |
| `gate status` | `gate` 或 `gate info` |

## 常见问题

**Q：设置代理后新开终端不生效？**  
A：`gate set` 只写入当前进程级。运行 `gate install-shell-hook` 写入 shell profile 实现持久化。

**Q：工具显示「未安装」但已安装？**  
A：可能是非标准安装路径。运行 `gate path -n <tool> --exec <path>` 手动指定。

**Q：如何只为 git 配置代理而不影响全局？**  
A：`gate app git http://...`，不运行 `gate set`。

**Q：preset load 后代理没有写入工具配置文件？**  
A：确认保存预设时已有工具代理配置。可用 `gate app --all http://...` 先批量配置再保存预设。

**Q：如何查看 gate 存储了哪些文件？**  
A：运行 `gate doctor --verbose`，或直接查看数据根目录。

**Q：代理格式是什么？**  
A：标准格式 `http://host:port` 或 `http://user:pass@host:port`。支持 `http://` 和 `https://` 协议。
